# PART A: 架构概述（人读）

## A.1 一句话定位

`ae dev-loop` 是一个 **Stage-sequenced Agent loop**：接收自然语言需求，按 architect → developer → critic 三阶段串行推进，每阶段由 LLM Agent 驱动工具调用，Guardrail + Gate 双层质量防护，SQLite Checkpoint 持久化支持中断恢复，Claude Code Plugin 形态承载 UX。

## A.2 核心概念（5 个）

| 概念 | 一句话定义 | 类比 |
|------|-----------|------|
| **Stage** | 三阶段流水线的一步 (architect/developer/critic) | LangGraph 的一个 Node |
| **Round** | Stage 的一次完整执行 = Agent 调用 + Guardrail 检查 + Gate 执行 | LangGraph 的一个 Superstep |
| **Guardrail** | 进程内确定性校验（快，4 态：pass/block/drop/retry） | CrewAI GuardrailResult |
| **Gate** | subprocess 质量检查（准，pass/fail）—— linter, type checker, tests 等 7 道 | CI Pipeline Stage |
| **Checkpoint** | 每 Round 结束时的完整状态快照（SQLite WAL） | LangGraph SqliteSaver |

## A.3 端到端执行流程（一次完整的 `/dev-loop "实现用户登录"`）

```
用户输入: /dev-loop "实现用户登录"
│
├─ [预检] 检查 ANTHROPIC_API_KEY、project state、uv
│
├─ Round 1: Stage=architect
│   ├─ Guardrail(pre): RequirementValid → pass
│   ├─ Agent: ArchitectAgent 分析需求 → 读代码 → 输出 plan + file_list
│   ├─ Guardrail(post): PlanExists → pass
│   ├─ Gates(parallel): safety, lint, type_check, test, build
│   └─ Checkpoint: 保存状态 → next_stage=developer
│
├─ Round 2: Stage=developer
│   ├─ Guardrail(pre): (none for developer)
│   ├─ Agent: DeveloperAgent 按 plan 执行 TDD → RED → GREEN → REFACTOR → commit
│   ├─ Guardrail(post): GitDiffExists → pass, TestsPass → pass, GitClean → pass
│   ├─ Gates(parallel): safety, lint, type_check, test, build
│   └─ Checkpoint: 保存状态 → next_stage=critic
│
├─ Round 3: Stage=critic
│   ├─ Guardrail(pre): (none for critic)
│   ├─ Agent: CriticAgent 审查 commit → 5 维度检查 → 判定 APPROVE/MAJOR
│   ├─ Guardrail(post): (none for critic)
│   ├─ Gates(parallel): safety, lint, type_check, test, build
│   └─ Judge: APPROVE? → GOAL_ACHIEVED → 停止
│
└─ 输出: { verdict: "APPROVE", rounds: 3, gate_results: [...] }
```

**若 critic 判 MAJOR**，Round 4 回到 Stage=developer，注入 critic_feedback，最多 3 次 MAJOR 回路。

## A.5 关键设计决策

| # | 决策 | 理由 | 业界参照 |
|---|------|------|---------|
| D1 | **Stage-sequenced rounds**（每轮一个 Stage，非全 Agent 并行） | architect→developer→critic 有硬依赖链，全并行无意义 | LangGraph PregelLoop: 每个 superstep 执行一类 node |
| D2 | **同一 Stage 内多 task 可并行** | 如多个 developer task 修改不同文件可并行 | LangGraph asyncio.gather in _run_tasks |
| D3 | **Guardrail 分 pre/post，按 stage 过滤** | 避免 GitDiffExists 在 architect 阶段误判 | CrewAI Guardrail: 按 task 类型注册 |
| D4 | **Gate 全并行 (asyncio.to_thread)** | 7 个 Gate 互不依赖，串行浪费 ~4min/round | CI 并行 job 模式 |
| D5 | **LoopState (运行时) + CheckpointEnvelope (持久化) 双容器** | 解耦运行效率与持久化格式 | LangGraph State (runtime) + Checkpoint (persistent) |
| D6 | **Plugin = Bash 委托 `uv run ae`** | Python 控制流 + Claude Code UX，不重写引擎 | Claude Code plugin convention |

### B1.3 LoopState → CheckpointEnvelope 转换契约

```
转换时机: Orchestrator._save_checkpoint() 内
转换逻辑:
  1. state_snapshot = {
       "requirement": state.requirement,
       "round": state.round,
       "stage": state.stage,
       "plan": state.plan,
       "file_list": state.file_list,
       "files_changed": state.files_changed,
       "commit_hash": state.commit_hash,
       "test_results": state.test_results,
       "verdict": state.verdict,
       "findings": state.findings,
       "critic_feedback": state.critic_feedback,
       "round_history": list(state.round_history)  # deque → list
     }
  2. envelope = CheckpointEnvelope(
       thread_id=thread_id,
       round=state.round,
       step=1,
       status="running" if not finished else "completed",
       state_snapshot=state_snapshot,
       channels={name: ch.copy() for name, ch in self.channels.items()},
       channel_versions=dict(self.channel_versions)
     )
  3. store.save(envelope) → checkpoint_id
  4. state.checkpoint_id = checkpoint_id

逆向转换 (load/resume):
  1. envelope = store.load(thread_id, round, step)
  2. state = LoopState(**envelope.state_snapshot)  # dict → dataclass
  3. state.round_history = deque(envelope.state_snapshot["round_history"], maxlen=50)
  4. 恢复 channels: 遍历 envelope.channels, from_checkpoint() 重建
```

### B1.10 SQLite DDL

```sql
-- Checkpoint 主表
CREATE TABLE IF NOT EXISTS checkpoints (
    thread_id   TEXT NOT NULL,
    round       INTEGER NOT NULL,
    step        INTEGER NOT NULL DEFAULT 1,
    status      TEXT NOT NULL DEFAULT 'running',
    state_json  TEXT NOT NULL,
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (thread_id, round, step)
) WITHOUT ROWID;

-- Gate 结果明细表
CREATE TABLE IF NOT EXISTS gate_results (
    thread_id   TEXT NOT NULL,
    round       INTEGER NOT NULL,
    gate_name   TEXT NOT NULL,
    passed      INTEGER NOT NULL,
    message     TEXT DEFAULT '',
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (thread_id, round, gate_name),
    FOREIGN KEY (thread_id, round) REFERENCES checkpoints(thread_id, round)
);

-- Round 历史明细表
CREATE TABLE IF NOT EXISTS round_history (
    thread_id   TEXT NOT NULL,
    round       INTEGER NOT NULL,
    stage       TEXT NOT NULL,
    files_changed INTEGER DEFAULT 0,
    lines_added INTEGER DEFAULT 0,
    lines_removed INTEGER DEFAULT 0,
    semantic_satisfied INTEGER,  -- 0/1/NULL
    tasks_run   TEXT NOT NULL DEFAULT '[]',
    task_outcomes TEXT NOT NULL DEFAULT '{}',
    guardrail_result TEXT,
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (thread_id, round)
);
```

**连接配置**: `PRAGMA journal_mode=WAL; PRAGMA busy_timeout=5000; PRAGMA foreign_keys=ON;`

### B2.2 StageGraph

```
StageGraph.next_stage(state: LoopState) → Literal["architect","developer","critic"] | None
  # 输入: 当前 LoopState (读 state.stage + state.verdict + state.round)
  # 输出: 下一个 stage 名称, 或 None (所有阶段完成)
  # 逻辑:
  #   state.stage == ""       → "architect"
  #   state.stage == "architect" → "developer"
  #   state.stage == "developer" → "critic"
  #   state.stage == "critic" + state.verdict == "APPROVE" → None
  #   state.stage == "critic" + state.verdict == "MAJOR" → "developer"
  # 无副作用 (纯函数)
```

### B2.6 CheckpointStore

```

SQLiteCheckpointStore.save(envelope: CheckpointEnvelope) → str

  # raises: OSError (disk full)
  #         RuntimeError (SQLite 异常, busy_timeout 后仍失败)

  # 返回: [{thread_id, round, step, status, created_at}, ...] (按 round DESC)

  # 返回: 所有已知 thread_id (去重, 按最新 created_at DESC)
```

### B3.2 转换表

| # | 从 stage | 到 stage | 条件 | 副作用 |
|---|---------|---------|------|--------|
| T1 | `""` (idle) | `"architect"` | CMD-01 触发 | round=1, 初始化 LoopState |
| T2 | `"architect"` | `"developer"` | Guardrail(post) all pass | plan + file_list 写入 state |
| T3 | `"developer"` | `"critic"` | Guardrail(post) all pass | commit_hash + test_results 写入 state |
| T4 | `"critic"` | `""` (idle) | verdict=="APPROVE" | Judge 触发 GOAL_ACHIEVED → stop |
| T5 | `"critic"` | `"developer"` | verdict=="MAJOR" AND retry_count < 3 AND 非连续2次MAJOR | critic_feedback 注入, retry_count++ |

### B3.3 next_stage() 实现逻辑

```
next_stage(state) → str | None:
  if state.stage == "":     return "architect"
  if state.stage == "architect": return "developer"
  if state.stage == "developer": return "critic"
  if state.stage == "critic":
    if state.verdict == "APPROVE": return None
    if state.verdict == "MAJOR":   return "developer"
    return None  # verdict 为空 (不应出现, 防御)
  return None
```

### B3.4 Stage 内执行约束

- 每个 Stage 对应 **1 个 Agent role**
- 同一 Stage 可执行 **1..N 个 Task**（N≤20）
- Agent 可进行 **1..10 轮 tool_use loop**
- Stage 间数据传递: **通过 LoopState 字段**（architect 写 plan → developer 读 plan）
- MAJOR 回路: developer 读取 `state.critic_feedback`，注入到 system prompt 末尾

## B5. Guardrail 详细规格

### B5.1 Guardrail 基类规格

```
Guardrail:
    name: str                        # 唯一标识
    timing: Literal["pre","post"]    # pre=Agent 执行前, post=Agent 执行后
    applies_to: Literal["architect","developer","critic"]  # 适用 Stage

    check(stage: str, state: LoopState) → GuardrailResult
```

### B5.2 5 个 Guardrail 逐个规格

| # | name | timing | applies_to | 检查内容 | 失败动作 | 重试上限 |
|---|------|--------|-----------|---------|---------|---------|
| G1 | **RequirementValid** | `"pre"` | `"architect"` | `state.requirement` 非空, 1..4096 chars, 不含仅控制字符 (0x00-0x1F 除 \n\t) | `block` | — |
| G2 | **PlanExists** | `"post"` | `"architect"` | `state.plan` 非空 AND `state.file_list` 非空 AND len(file_list) ≥ 1 | `retry` | 3 |
| G3 | **GitDiffExists** | `"post"` | `"developer"` | `git diff HEAD~1..HEAD --numstat` 输出非空 | `retry` | 3 |
| G4 | **TestsPass** | `"post"` | `"developer"` | `state.test_results["failed"] == 0` AND `state.test_results["errors"] == 0` | `retry` | 3 |
| G5 | **GitClean** | `"post"` | `"developer"` | `git status --porcelain` 输出为空 (无未提交变更) | `block` | — |

### B5.3 GuardrailChain 完整逻辑

```
GuardrailChain:
    guardrails: list[Guardrail]  # 构造时注入, 按优先级排序

    check(timing, stage, state) → GuardrailResult:
        for g in self.guardrails:
            if g.timing != timing:      # 过滤: timing 不匹配 → 跳过
                continue
            if g.applies_to != stage:   # 过滤: stage 不匹配 → 跳过
                continue
            result = g.check(stage, state)
            if result.action != "pass":
                return result  # fail-fast: 第一个失败就返回
        return GuardrailResult("pass", "all checks passed")
```

**注册顺序 (构造时注入)**:
1. RequirementValid (pre-architect)
2. PlanExists (post-architect)
3. GitDiffExists (post-developer)
4. TestsPass (post-developer)
5. GitClean (post-developer)

### B5.4 Guardrail 响应处理 (在 Orchestrator._handle_guardrail_result)

```
_handle_guardrail_result(result, stage, retry_count) → "continue" | "stop":
    switch result.action:
        case "pass":
            return "continue"
        case "block":
            state.verdict = ""  # 清空 verdict
            state.round_history[-1].guardrail_result = "block"
            return "stop"  # 停止循环, 输出错误
        case "drop":
            # 丢弃当前 stage 产出, 清空对应字段
            _clear_stage_fields(state, stage)
            retry_count += 1
            return "continue"  # 重跑同一 stage (不减 max_rounds)
        case "retry":
            if retry_count >= 3:
                return _handle_guardrail_result(
                    GuardrailResult("block", f"retry 超限 ({retry_count}/3)"),
                    stage, retry_count
                )
            retry_count += 1
            return "continue"  # 重跑同一 stage
```

**_clear_stage_fields 逻辑**:
- stage="architect" → 清空 `state.plan`, `state.file_list`
- stage="developer" → 清空 `state.files_changed`, `state.commit_hash`, `state.test_results`
- stage="critic" → 清空 `state.verdict`, `state.findings`, `state.critic_feedback`

## B6. Gate 详细规格

### B6.1 7 道 Gate 逐个规格

| # | gate_name | 实现方式 | 输入 | 判定 | 耗时(典型) | 依赖 | 超时 |
|---|-----------|---------|------|------|----------|------|------|
| G1 | `safety` | 正则扫描源码文件 | `project_root/**/*.py` 等 | 匹配秘密模式 → `failed` | <5s | 无 | 30s |
| G2 | `lint` | `subprocess.run([linter_cmd])` | `project_root` | exit≠0 → `failed` | 10-30s | 无 | 120s |
| G3 | `type_check` | `subprocess.run([type_checker_cmd])` | `project_root` | exit≠0 → `failed` | 10-30s | 无 | 120s |
| G4 | `contract` | 检查 architect.contracts vs 实际代码 | `project_root` | 多 Agent 模式 + contracts 未满足 → `failed` | <5s | lint | 30s |
| G5 | `test` | `subprocess.run([test_runner_cmd])` | `project_root` | exit≠0 → `failed` | 30-120s | 无 | 180s |
| G6 | `coverage` | 永远返回 `skip` | — | 16G 内存约束, 冻结 | 0s | — | — |
| G7 | `build` | `subprocess.run([build_cmd])` | `project_root` | exit≠0 → `failed` | 10-60s | lint, type_check | 120s |

### B6.2 Gate 并发调度

```
run_gates(gates, project_root) → dict[str, Verdict]:
    if not gates or project_root is None:
        return {}

    # 所有 Gate 并行启动 (asyncio.to_thread)
    coros = [_run_one_gate(g, project_root) for g in gates]
    gathered = await asyncio.gather(*coros, return_exceptions=True)

    results = {}
    for item in gathered:
        if isinstance(item, BaseException):
            continue  # 理论不应出现 (_run_one_gate 内部捕获)
        name, verdict = item
        results[name] = verdict
    return results

_run_one_gate(gate, project_root) → tuple[str, Verdict]:
    try:
        verdict = await asyncio.to_thread(gate.run, project_root)
    except Exception as e:
        verdict = Verdict.failed(f"Gate {gate.name} 异常: {e}", gate_name=gate.name)
    return (gate.name, verdict)
```

- **并行模式**: 7 个 Gate 同时启动, 总耗时 ≈ max(单个) 而非 sum
- **依赖 Gate**: ContractGate 依赖 LintGate. 并行下 ContractGate 自行检查 lint 结果 (从 gate_results dict 读取, 若 lint 尚未执行完 → 等待)
- **异常处理**: 单个 Gate 异常不传播, 转为 `Verdict.failed`
- **超时**: 每个 Gate 内部 `subprocess.run(timeout=N)` 控制

### B6.3 Gate 执行时机

Gate 在每轮 Agent 执行完成后、Judge 判定前运行。Run_round 内部执行顺序:
1. Agent tasks (asyncio.gather)
2. Gates (asyncio.gather)
3. 构造 RoundHistory (含 gate_results)
4. 返回 RoundResult

## B7. Channel 系统

### B7.1 设计定位

Channel 系统 = checkpoint 持久化专用的状态存储抽象层。运行时状态走 `LoopState` dataclass (Python 原生字段访问, 零开销)。Channel 仅用于 `CheckpointEnvelope.channels` — 需要跨 session 持久化且类型安全的状态。

### B7.2 协议定义 (LangGraph BaseChannel 对齐)

```
BaseChannel[Value, Update, Checkpoint]:
    # 序列化三件套 (LangGraph 协议)
    copy() → Self                          # 深拷贝当前 channel
    checkpoint() → Checkpoint              # 序列化为可持久化的值
    from_checkpoint(c: Checkpoint) → Self  # 从持久化值重建 channel 实例

    # 读写接口
    get() → Value                          # 读当前值 (空时 raise EmptyChannelError)
    update(values: Sequence[Update]) → bool  # 批量更新. True=channel 值有变化
    is_available() → bool                  # 是否有值可读 (不抛异常)

    # 元数据
    key: str                               # channel 名称
    typ: type                              # 值类型标记
```

### B7.3 Channel 类型清单

| Channel 类 | Value 类型 | Update 类型 | Checkpoint 类型 | 语义 |
|-----------|-----------|------------|----------------|------|
| `LastValue[T]` | `T` | `T` | `T` | 存最后写入的值, 新值覆盖旧值 |
| `BarrierChannel[T]` | `list[T]` | `T` | `list[T]` | 收集 N 个 task 的输出, 全部到齐后才可读 |
| `Topic[T]` | `list[T]` | `tuple[str, T]` | `list[tuple[str, T]]` | 按 topic 分组存储 (预留 Send API) |

### B7.4 Channel 版本追踪

- 每个 channel 维护内部 version 计数器 (int, 从 0 开始)
- `update()` 返回 True → version += 1
- `channel_versions` dict 存储每个 channel 的当前 version
- 用途:
  1. Checkpoint 差异检测: `_get_new_channel_versions(prev, curr) → set[str]` — 找出被修改的 channel
  2. 增量 task 触发: 只有关联 channel 被修改的 task 才在下一轮执行 (参考 LangGraph `_prepare_next_tasks`)
  3. 停滞检测辅助: 无任何 channel version 变化 → 可能停滞

### B8.1 完整 run() 流程

```
Orchestrator.run(cancellation?) → list[RoundHistory]:

  │ max_rounds = judge.config.max_iterations                  │
  │ retry_counter = {stage: 0}  (per-stage retry 计数)        │
  └──────────────────────────────────────────────────────────┘

  ┌─ 2. 主循环 (for round_id in 1..max_rounds) ────────────┐
  │                                                          │
  │ 2a. cancellation?.check() → break                        │
  │                                                          │
  │ 2b. current_stage = state.stage                          │
  │     if current_stage is None: break  # 所有阶段完成       │
  │                                                          │
  │ 2c. round_tasks = Plan.get_tasks_by_stage(current_stage) │
  │     只取 role == current_stage 的 task                    │
  │                                                          │
  │ 2d. ★ Guardrail PRE check ──────────────────────────    │
  │     pre_result = guardrail_chain.check("pre",            │
  │                    current_stage, state)                  │
  │     action = _handle_guardrail_result(pre_result,        │
  │               current_stage, retry_counter)               │
  │     if action == "stop": break                            │
  │     if action == "retry": continue  # 重跑同一 stage      │
  │     if action == "drop": continue   # 重跑同一 stage      │
  │                                                          │
  │ 2e. ★ Agent 执行 ───────────────────────────────────    │
  │     round_result = await run_round(                      │
  │         stage=current_stage,                              │
  │         tasks=round_tasks,                                │
  │         executor=executor,                                │
  │         ctx=state,                                        │
  │         cancellation=cancellation,                        │
  │         round_id=round_id,                                │
  │         gates=gates,                                      │
  │         project_root=project_root)                        │
  │     → 将 Agent 输出写入 state 对应字段                     │
  │                                                          │
  │ 2f. ★ Guardrail POST check ─────────────────────────    │
  │     post_result = guardrail_chain.check("post",          │
  │                     current_stage, state)                 │
  │     action = _handle_guardrail_result(post_result,       │
  │               current_stage, retry_counter)               │
  │     if action == "stop": break                            │
  │     if action == "retry": continue                        │
  │     if action == "drop": continue                         │
  │                                                          │
  │ 2g. ★ Semantic 评估 ────────────────────────────────    │
  │     semantic = await semantic_evaluator(round_result)     │
  │     round_result.history_entry.semantic_satisfied =       │
  │       semantic                                            │
  │                                                          │
  │ 2h. state.round_history.append(                          │
  │       round_result.history_entry)                         │
  │                                                          │
  │ 2i. ★ ★ ★ 先保存 Checkpoint ★ ★ ★ ──────────────────   │
  │     _save_checkpoint(state)  # 先保存, 再判断             │
  │     # 确保每轮 (含最后一轮) 都有 checkpoint                │
  │                                                          │
  │ 2j. ★ Judge 判定 (在 checkpoint 之后) ──────────────    │
  │     verdict = judge.evaluate(history=state.round_history) │
  │     if verdict.should_stop:                               │
  │         return list(state.round_history)                  │
  │                                                          │
  │ 2k. ★ Stage 推进 ──────────────────────────────────     │
  │     next_stage = StageGraph.next_stage(state)             │
  │     if next_stage is None: break                          │
  │     state.stage = next_stage                              │
  │     retry_counter[next_stage] = 0  # 重置重试计数         │
  │     state.round += 1                                      │
  └──────────────────────────────────────────────────────────┘

  ┌─ 3. 退出 ─────────────────────────────────────────────┐
  │ verdict = Verdict.stop(level=4,                          │
  │     reason=f"达到最大轮次 {max_rounds}")                  │
  │ return list(state.round_history)                         │
  └──────────────────────────────────────────────────────────┘
```

## B9. 收敛判定

### B9.1 4 级判定

```
ConvergenceJudge.evaluate(history) → Verdict:
    # 优先级: 硬上限 > 质量门 > 停滞 > 语义

    # Level 4: 硬上限
    if len(history) >= config.max_iterations:
        return Verdict.stop(4, f"达到最大迭代次数 {config.max_iterations}")

    # Level 3: 质量门
    latest = history[-1]
    if latest.gate_results:
        failed = [name for name, v in latest.gate_results.items() if not v.passed]
        if not failed:
            return Verdict.stop(3, f"所有质量门通过 ({len(latest.gate_results)} 道)")
        else:
            return Verdict.stop(3, f"质量门失败 ({len(failed)} 道): {', '.join(failed)}")

    # Level 2: 停滞检测
    if detect_stagnation(history, config.stagnation_threshold, config.stagnation_diff_ratio):
        return Verdict.stop(2, f"连续 {config.stagnation_threshold} 轮无实质变化")

    # Level 1: 语义收敛
    if latest.semantic_satisfied is True:
        return Verdict.stop(1, "LLM 评估: 本轮产出满足需求")

    # 默认: 继续
    return Verdict.continue_()
```

### B9.2 停滞检测算法

```
detect_stagnation(history, threshold=2, diff_ratio_threshold=0.05) → bool:
    if len(history) < threshold + 1:
        return False
    consecutive = 0
    for i in range(len(history) - 1, 0, -1):
        ratio = diff_ratio(history[i], history[i-1])
        if ratio < diff_ratio_threshold:
            consecutive += 1
            if consecutive >= threshold:
                return True
        else:
            consecutive = 0
    return False

diff_ratio(curr, prev) → float:
    # 4 维度: files_changed + lines_added + lines_removed
    curr_size = curr.files_changed + curr.lines_added + curr.lines_removed
    prev_size = prev.files_changed + prev.lines_added + prev.lines_removed
    if curr_size == 0 and prev_size == 0:
        return 0.0
    return abs(curr_size - prev_size) / max(curr_size, prev_size)
```

## B10. Checkpoint 系统

### B10.1 保存流程

```
_save_checkpoint(state) → str:
    1. snapshot = {
         "requirement": state.requirement,
         "round": state.round,
         "stage": state.stage,
         "plan": state.plan,
         "file_list": state.file_list,
         "files_changed": state.files_changed,
         "commit_hash": state.commit_hash,
         "test_results": state.test_results,
         "verdict": state.verdict,
         "findings": state.findings,
         "critic_feedback": state.critic_feedback,
         "round_history": [asdict(h) for h in state.round_history]
       }
    2. envelope = CheckpointEnvelope(
         thread_id=thread_id,
         round=state.round,
         step=1,
         status="running",
         state_snapshot=snapshot,
         channels={k: ch.copy() for k, ch in self._channels.items()},
         channel_versions=dict(self._channel_versions)
       )
    3. checkpoint_id = store.save(envelope)  # 返回 thread_id
    4. state.checkpoint_id = checkpoint_id
    5. 同时写入 gate_results 表和 round_history 表 (在同一 SQLite tx 内)
```

### B10.2 加载/恢复流程

```
resume(thread_id, round, step) → LoopState:
    1. envelope = store.load(thread_id, round, step)
    2. state = LoopState(**envelope.state_snapshot)
    3. state.round_history = deque(
         envelope.state_snapshot.get("round_history", []),
         maxlen=50
       )
    4. 恢复 channels:
       for name, ch_data in envelope.channels.items():
           ch_type = _get_channel_type(name, ch_data)  # 从类型注册表推断
           self._channels[name] = ch_type.from_checkpoint(ch_data)
    5. return state  # 可以继续执行
```

### B10.3 Crash 恢复场景

| 场景 | 检测 | 恢复动作 |
|------|------|---------|
| Round K 中途 OS kill | checkpoint 最新 = K-1 | resume → 重跑 Round K |
| Round K SQLite tx 中断 | WAL 自动回滚未提交 tx | resume → 重跑 Round K |
| Checkpoint JSON 损坏 | `json.JSONDecodeError` | 返回 `{"__raw__": raw_str}` → 提示用户 |
| Schema 版本不匹配 | `_migrate_checkpoint` 钩子 | 尝试字段映射 → 失败则 raise |
| Disk full | `OSError` at save | abort, 提示释放空间 |
| SQLite busy | `busy_timeout=5000ms` 耗尽 | raise `RuntimeError` → caller 决定 |

## B11. 并发与线程模型

### B11.1 三层模型

```
Layer 1: asyncio event loop (主线程, 1 个)
  - Orchestrator.run() 是 async coroutine
  - run_round() 内 asyncio.gather 调度 task
  - run_gates() 内 asyncio.gather 调度 gate
  - 所有阻塞 I/O 通过 asyncio.to_thread 移到 thread pool

Layer 2: ThreadPoolExecutor (asyncio 默认)
  - Anthropic SDK 同步 create_message → asyncio.to_thread
  - Gate subprocess.run → asyncio.to_thread
  - git diff --numstat → asyncio.to_thread
  - 线程池大小: min(32, os.cpu_count() + 4)

Layer 3: Subprocess (Gate 工具调用)
  - 每个 Gate 独立进程 (linter, type checker, tests)
  - timeout 控制: subprocess.run(timeout=N)
  - 取消: process.terminate() → SIGTERM → 5s → SIGKILL
```

### B11.2 资源约束

| 资源 | 上限 | 机制 |
|------|------|------|
| 并发 Agent | 按 Stage 串行, 同 Stage 内 ≤20 task 并行 | Plan.get_tasks_by_stage() |
| 并发 Gate | 7 个全部并行 | asyncio.gather + asyncio.to_thread |
| LLM 并发请求 | 1 per Agent (单 Agent 执行) | Anthropic SDK 管理 |
| SQLite 连接 | 1 写连接 (WAL mode) | busy_timeout=5000ms |
| RoundHistory 内存 | 50 条 | deque(maxlen=50) |
| Checkpoint 磁盘 | WAL 文件 < 10MB | 定期 VACUUM |

### B11.3 取消协议

```
CancellationToken:
    is_cancelled() → bool
    check() → raise AEError(TASK_CANCELLED) | None

取消安全点:
    - Round 开始前 (2a)
    - Stage 边界 (2k 之后)
    不在 Agent LLM 调用中插入取消 (等当前调用返回)
    不在 Gate subprocess 中插入取消 (等 subprocess 超时)
```

### B12.2 LLM 重试参数 (exponential backoff)

| 参数 | 值 | 说明 |
|------|-----|------|
| base_delay | 1.0s | 首次重试等待 |
| max_delay | 30.0s | 重试等待上限 |
| multiplier | 2.0 | 每次重试 delay *= multiplier |
| jitter | ±25% | 随机抖动 (避免 thundering herd) |
| max_retries | 3 | 总重试次数上限 |

**计算**: delay = min(base_delay * multiplier^(retry-1) * random(0.75, 1.25), max_delay)

### B12.3 降级路径完整矩阵

| 场景 | 降级动作 | 影响 |
|------|---------|------|
| SafetyGate 工具缺失 → skip | 无 secret 检测 | 低风险 (pre-tool hook 有独立 sandbox) |
| CoverageGate → 永远 skip | 覆盖率由 CI 独立检查 | 设计决策 (16G 约束) |
| SemanticEvaluator 不可用 → None | 仅用停滞检测 (level 2) | 第 1 级语义收敛失效 |
| Gate subprocess timeout → failed | 该 gate 判定失败 | Judge 可能误停 |
| checkpoint.save 失败 → abort | 本轮进度丢失 | 可从上一 checkpoint resume |
| post-edit hook 异常 → catch + exit 0 | 无 secret 检测 | 不影响主流程 |
| pre-tool hook 超时 (>5s) → 放行 | 安全检查失效 | 安全优先于速度的权衡 |
| SemanticEvaluator LLM 异常 → None | 降级到停滞检测 | 同上 |

### B17.1 5 框架 × 13 维度详细对比

### B17.2 v5.0 差异化优势

- **Stage-sequenced + per-stage parallel**: 比 LangGraph 全并行更可预测 (architect 必定在 developer 之前), 比 CrewAI 全串行更高效 (同 stage 内可并行)
- **双层验证**: Guardrail (进程内, <10ms) + Gate (subprocess, 精确) — LangGraph 只有人工 interrupt, AutoGen 只有回调
- **Init ↔ Loop 共享契约**: `.ae-answers.yml` — 业界无类比
- **Plugin 首创**: Claude Code 原生 slash command + hook 集成

## B20. 验收标准 (EARS 格式)

| ID | 验收条件 |
|----|---------|

| AC-02 | When 用户按 Ctrl-C 中断 dev-loop, the system shall 保存当前 checkpoint 并 exit 130 |
| AC-03 | When 用户输入 `/checkpoint resume <thread_id>`, the system shall 从最近 checkpoint 恢复状态并继续未完成的 Stage |
| AC-04 | When post-edit hook 检测到代码变更, the system shall 在 30s 内完成 SafetyGate 扫描 |
| AC-05 | When pre-tool hook 检测到危险 Bash 命令 (匹配 13 denylist 模式), the system shall 拒绝执行并 exit 2 |
| AC-06 | While Stage=critic, when developer 提交有 P0 问题的代码, the critic shall 判定 MAJOR 并给出 file:line level 的具体修复建议 |
| AC-07 | While 连续 2 轮 diff_ratio < 5%, the ConvergenceJudge shall 触发 STAGNANT 并停止循环 |
| AC-08 | When lint Gate exit≠0, the Judge shall 返回 level=3 QUALITY_FAIL 并含具体错误信息 |
| AC-09 | When ANTHROPIC_API_KEY 未设置, the system shall 在 CMD-01 预检阶段 exit 1 |
| AC-10 | When 用户输入 `/status`, the system shall 返回 thread_id, round, stage, verdict, 近 5 轮 history 摘要 |
| AC-11 | While Stage=architect, when GuardrailChain.check("post","architect",state) 被调用, the GitDiffExists guardrail shall 因 applies_to!="architect" 被过滤跳过 |
| AC-12 | When 任一 Stage 完成, the checkpoint shall 在 Judge 判定之前保存 (确保最后一轮也有 checkpoint) |
