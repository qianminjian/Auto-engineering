## 一、v2.0 与 v1.0 关系

| 维度 | v1.0（已实现） | v2.0（新增） |
|------|---------------|-------------|
| 模块 | `engine/` + `runtime/` + `tools/` | `loop/` + `gates/`（叠加，不替代） |
| 循环 | `while True + tick/after_tick` | `Round.run + asyncio.gather` |
| 状态 | `DevLoopState`（混 state+checkpoint） | `LoopState`（dataclass）+ `Channel` 系统 + SQLite `Checkpoint` 分离 |
| 多 Agent | 串行 | `TaskDAG + check_file_isolation` → `asyncio.gather` 并发 |
| 收敛 | 无 | `ConvergenceJudge` 4 级（硬上限/质量/停滞/语义） |
| 质量门 | Guardrail 4 态 | 7 道 Gate（safety/lint/type_check/contract/test/coverage/build） |
| CLI | `ae init / ae dev-loop / ae status` | v2 增量：`ae checkpoint v2 list/show/delete` + `ae status` 增强 |

> **关键决策（决策 11/12）**：v2.0 是**增量式**，不删除 v1.0 模块。CLI 仍 import `engine/runtime/tools`（旧路径兼容），`loop/` 是 v2.0+ 首选路径。

## 二、Loop 子系统架构（v2.0）

```
┌───────────────────────────────────────────────────────────────┐
│                v2.0 Loop 子系统（auto_engineering/loop/）       │
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │              Orchestrator（主循环）                        │ │
│  │   Orchestrator.run() ── while round < max_rounds:         │ │
│  │     plan = PlanBuilder.build(requirement)                 │ │
│  │     check_file_isolation(plan)         ← 确定性            │ │
│  │     result = run_round(round, tasks)   ← asyncio.gather   │ │
│  │     verdict = ConvergenceJudge.judge(history)             │ │
│  │     save_checkpoint(LoopState, round)                     │ │
│  │     if verdict == GOAL_ACHIEVED: break                   │ │
│  └──────┬───────────────────────────────────────────────────┘ │
│         │                                                       │
│  ┌──────▼───────────────────────────────────────────────────┐ │
│  │        L1: Round 生命周期（loop/round.py）                │ │

│  │     executor.execute(task_2) ──┤ 共享 LoopState Channel   │ │
│  │     executor.execute(task_3) ──┘                          │ │
│  │   )                                                       │ │
│  │   → RoundResult (task_outcomes, gate_results)             │ │
│  └──────┬───────────────────────────────────────────────────┘ │
│         │                                                       │
│  ┌──────▼───────────────────────────────────────────────────┐ │
│  │        L2: Plan + 拓扑 + 文件隔离（loop/plan.py）          │ │
│  │   TaskDAG(tasks) ── topological_sort()                   │ │
│  │                    ── check_file_isolation()              │ │
│  │   Plan(round, tasks, dag, parallel_groups)                │ │
│  └──────┬───────────────────────────────────────────────────┘ │
│         │                                                       │
│  ┌──────▼───────────────────────────────────────────────────┐ │
│  │        L3: State + Channel 系统（loop/state.py）          │ │
│  │   LoopState(Pydantic BaseModel)                          │ │
│  │     ├── LastValueChannel[T]    ← Plan/Review 状态         │ │
│  │     ├── AccumulatingChannel[T] ← Task 完成列表/Gate 结果  │ │
│  │     └── BarrierChannel         ← 多 Agent 完成同步点      │ │
│  └──────┬───────────────────────────────────────────────────┘ │
│         │                                                       │
│  ┌──────▼───────────────────────────────────────────────────┐ │

│  │   SQLiteCheckpointStore   ← 事务 + Schema 版本控制       │ │
│  │   ConvergenceJudge        ← 4 级（硬上限/质量/停滞/语义）  │ │
│  └──────────────────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────────────────┘
```

## 三、核心数据结构

### 3.1 Channel 系统（loop/state.py）

```python
class Channel[T](ABC):
    """Channel 基类 — 泛型 channel 抽象（借鉴 LangGraph）"""

class LastValueChannel(Channel[T]):
    """单写, 后续覆盖 (Plan 状态、Review 结论)"""

class AccumulatingChannel(Channel[T]):
    """多写, append 列表 (Task 完成列表、Gate 结果汇总)"""

class BarrierChannel(Channel[Any]):
    """等待所有 Agent 完成 (asyncio.Event 同步点)"""

class LoopState(BaseModel):
    """v2.0 运行时状态容器"""
    round: int
    step: int
    tasks: dict[str, Task]                  # LastValue
    task_results: dict[str, TaskOutcome]    # Accumulating
    gate_results: dict[str, GateResult]     # LastValue
    signals: list[Signal]                   # Topic
    metrics: MetricsSnapshot                # BinaryOperatorAggregate
    status: LoopStatus
```

### 3.2 Task DAG + 文件隔离（loop/plan.py）

```python
class TaskStatus(StrEnum):
    PENDING = "pending"
    IN_PROGRESS = "in_progress"
    COMPLETED = "completed"
    BLOCKED = "blocked"

@dataclass
class Task:
    id: str
    title: str
    description: str
    expected_output: str
    role: str                              # developer | critic | reviewer
    deps: list[str]                        # 前置 Task ID
    target_files: list[str]                # 涉及的文件（隔离键）
    context_files: list[str]
    validation: TaskValidation
    estimated_minutes: int
    status: TaskStatus
    output: TaskOutcome | None

@dataclass
class TaskDAG:
    tasks: dict[str, Task]
    edges: dict[str, list[str]]            # task_id → 依赖列表

def topological_sort(tasks: Iterable[Task]) -> list[str]: ...
def check_file_isolation(tasks: Iterable[Task]) -> list[str]:
    """确定性检查：返回冲突列表。冲突 → 串行或再拆分。"""

@dataclass
class Plan:
    round_number: int
    goal: str
    tasks: list[Task]
    task_dag: TaskDAG
    parallel_groups: list[list[str]]       # 拓扑排序后的并行组
    created_at: datetime
```

### 3.3 收敛判定（loop/convergence.py）

```python
class ConvergenceConfig:
    max_rounds: int = 10
    max_duration_seconds: int = 7200
    stagnation_threshold: int = 2
    stagnation_diff_ratio: float = 0.1
    quality_threshold: float = 1.0          # 所有 Gate 通过

class Verdict(StrEnum):
    CONTINUE = "continue"
    GOAL_ACHIEVED = "goal_achieved"          # 质量阈值通过
    STAGNANT = "stagnant"                    # 停滞检测
    MAX_ROUNDS = "max_rounds"                # 硬上限
    SEMANTIC = "semantic"                    # 语义收敛

class ConvergenceJudge:
    def judge(self, history: RoundHistory) -> Verdict:
        """4 级判定（顺序检查）:
        1. 硬上限：round >= max_rounds OR duration > max
        2. 质量阈值：所有 P0 criteria + 所有 Gate 通过
        3. 停滞检测：连续 stagnation_threshold 轮 diff_ratio < threshold
        4. 语义收敛：Orchestrator 评估"本轮产出与上轮相比是否有实质性改进"
        默认：CONTINUE
        """
```

### 3.4 Checkpoint 持久化（loop/checkpoint.py）

```python
SCHEMA_VERSION = 1

@dataclass
class CheckpointMeta:
    id: str
    round: int
    step: int
    parent_id: str | None
    created_at: datetime

@dataclass
class Checkpoint:
    meta: CheckpointMeta
    state: LoopState                        # 序列化 JSON

class SQLiteCheckpointStore:
    """SQLite 持久化 + 事务保证 + Schema 版本控制"""
    def save(self, state: LoopState, round: int, step: int) -> str: ...
    def load(self, checkpoint_id: str) -> Checkpoint: ...
    def list(self, round: int | None = None) -> list[CheckpointMeta]: ...
    def resume(self, checkpoint_id: str) -> LoopState:
        """拒绝 status == done 的 checkpoint；要求 status 持久化到 DB"""
```

## 四、7 道 Gate 系统（auto_engineering/gates/）

```
auto_engineering/gates/
├── __init__.py
├── base.py              ← Gate 基类 + GateResult + Verdict (pass/block/warn/skip)
├── safety.py            ← Gate 0: Semgrep + Gitleaks (Python AST 启发式)
├── lint.py              ← Gate 1: ruff check
├── type_check.py        ← Gate 2: mypy / pyright
├── contract.py          ← Gate 3: 跨 Agent 契约 (单 Agent 时 skip)
├── test.py              ← Gate 4: pytest
├── coverage.py          ← Gate 5: pytest --cov (≥80%)
└── build.py             ← Gate 6: 项目级 build 验证
```

### 4.1 Gate 基类（base.py）

```python
@dataclass
class GateResult:
    gate_name: str
    passed: bool
    blocked: bool
    details: str
    error_code: str | None

class Verdict(StrEnum):
    PASS = "pass"
    BLOCK = "block"
    WARN = "warn"
    SKIP = "skip"

class Gate(ABC):
    name: ClassVar[str]
    description: ClassVar[str]

    @abstractmethod
    async def run(self, project_root: Path, context: dict) -> GateResult: ...
    def should_skip(self, context: dict) -> bool: ...   # 默认 False
```

### 4.2 Gate 3（Contract）占位策略

- 单 Agent 场景：`should_skip(context)` 返回 True → 跳过
- 多 Agent 场景：检查跨角色契约文件（`design/contracts/*.md`）一致性
- 当前实现：占位（不检查具体契约内容，结构化返回 GateResult）

## 七、v2.0-Analysis-Loop.md §八 删除项取消说明

> **决策 11/12（BEACON.md）**：v2.0 实际采取增量路径，不删除 engine/runtime/tools 三层。

### 7.1 取消原因

| 原计划 | 实际 | 原因 |
|--------|------|------|
| 删 `engine/` (4 files) | 保留 | v1.1 实际工作非 stub；CLI 仍 `from auto_engineering.engine import LoopEngine` |
| 删 `crew/` (3 files) | 已不存在 | v1.1 Phase 0 已清理（BEACON 决策 7） |
| 删 `runtime/` (3 files) | 保留 | v1.1 实际工作非 stub；CLI 仍 `from auto_engineering.runtime.runtime import AgentRuntime` |
| 删 `tools/` (5 files) | 保留 | v1.1 实际工作非 stub；CLI 仍 `from auto_engineering.tools.* import ...` |

### 7.2 共存策略

- `engine/` 提供 `LoopEngine + build_dev_loop_graph`（v1.0 主循环）
- `loop/` 提供 `Orchestrator + Round + Channel`（v2.0 多 Agent 并发）
- 用户选择路径：`ae dev-loop` 仍走 v1.0（CLI 旧路径），新场景调用 `loop/` API 走 v2.0
- 未来迁移：v3.0 计划统一为单 loop/ 模块（v2.0 不在范围）

### 7.3 何时考虑真正删除

- v2.0 充分验证后（用户接受度高）
- v3.0 迁移完成（loop/ API 与 engine/ API 完全对齐）
- 引入替代 CLI 命令（移除旧 CLI 入口）

## 十、风险与缓解（v2.0 特有）

| 风险 | 概率 | 影响 | 缓解 |
|------|:---:|------|------|
| 多 Agent 文件集冲突（check_file_isolation 漏检） | 中 | 高 | DAG 构建时确定性检查 + 拓扑并行组隔离 + Git 原子提交兜底 |
| SQLite Checkpoint 状态不一致 | 低 | 高 | 事务保证 + Schema 版本号 + resume 状态校验 |
| asyncio.gather 异常传播 | 中 | 中 | TaskOutcome 单独记录失败状态；Orchestrator 主循环捕获 |
| Gate 3（Contract）误报/漏报 | 高 | 中 | 当前占位（should_skip=True），不阻塞主流程 |
| CLI v2 与 v1 命令冲突 | 低 | 低 | v2 命令加 `v2` 子命令（`checkpoint v2 list`），不破坏旧路径 |
