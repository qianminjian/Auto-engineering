## TDD 执行纪律

按 §1.3 + engineering-practices.md §1.3:
1. **Red**: 写测试 → 跑确认 FAIL
2. **Green**: 写最少实现 → 跑确认 PASS
3. **Refactor**: 重构保持测试绿
4. **commit**: atomic commit(每 P1 一个 commit 或相关合并)
5. **自动化**: 本会话完成 P1.1 + P1.2 + P1.5(本 session 阻断 dev-loop 项),剩 P1.3-P1.8 + P1.6 下个 session
