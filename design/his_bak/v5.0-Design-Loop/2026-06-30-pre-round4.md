# PART A: 架构概述（人读）

## A.1 一句话定位

`ae dev-loop` 是 **Stage-sequenced Agent loop**：接收自然语言需求 + 预定义 Task 列表，按 architect → developer → critic 三阶段串行推进，每阶段由 LLM Agent 驱动工具调用，Guardrail + Gate 双层质量防护，SQLite Checkpoint 持久化，Claude Code Plugin 承载 UX。

## A.2 核心概念

| 概念 | 定义 | 类比 |
|------|------|------|
| **Stage** | 三阶段流水线的一步 (architect/developer/critic)，每 Stage 执行同 role 的所有 Task | LangGraph Node |
| **Round** | 一个 Stage 的完整执行周期 = Agent 调用 + Guardrail + Gate + Judge + Checkpoint | LangGraph Superstep |
| **Task** | 工作单元，含 role/description/expected_output/depends_on/file_targets | AutoGen Task |
| **Guardrail** | 进程内确定性校验，4 态 (pass/block/drop/retry)，按 timing+stage 过滤 | CrewAI GuardrailResult |
| **Gate** | subprocess 质量检查，pass/fail，7 道并行执行 | CI Pipeline Stage |
| **MAJORRetryTracker** | 追踪 critic→developer 回路次数，防止无限循环 | LangGraph recursion_limit |
| **StageRouter** | 决定下一 Stage 的组件，综合 stage+verdict+MAJOR 计数判定 | LangGraph conditional edge |
| **Agent→LoopState 隔离** | Agent 不直接写 LoopState；Agent 返回 TaskOutcome，Orchestrator 提取写入 | Actor model message passing |

## A.3 端到端执行流程

```
用户输入: /dev-loop "实现用户登录" --tasks tasks.yml

┌─ 预检 ──────────────────────────────────────────────────┐
│ 检查 ANTHROPIC_API_KEY, project state, uv               │
│ 从 tasks.yml 加载 Task 列表 → Plan.validate()            │
│ Task 来源: 外部定义 (CLI --tasks / .ae-tasks.yml /       │
│   无文件时 ArchitectAgent 生成 batch_plan → 自动创建)     │
└──────────────────────────────────────────────────────────┘
│
├─ Round 1: Stage=architect
│   ├─ Guardrail(pre): RequirementValid → pass
│   ├─ Agent: ArchitectAgent → 产出 plan + file_list + batch_plan
│   ├─ (若 Tasks 为空) batch_plan → 自动创建 Task 列表
│   ├─ Guardrail(post): PlanExists → pass
│   ├─ Gates(parallel): safety, lint, type_check, test, build
│   ├─ Judge: CONTINUE
│   └─ Checkpoint: 保存 → StageRouter → next=developer
│
├─ Round 2: Stage=developer
│   ├─ Agent: DeveloperAgent × N tasks (asyncio.gather 并行)
│   │   每个 Agent 返回 TaskOutcome → Orchestrator 写入 state
│   ├─ Guardrail(post): GitDiffExists ✓, TestsPass ✓, GitClean ✓
│   ├─ Gates(parallel): 5 道
│   ├─ Judge: CONTINUE
│   └─ Checkpoint: 保存 → StageRouter → next=critic
│
├─ Round 3: Stage=critic
│   ├─ Agent: CriticAgent → 5 维度审查 → 判定 APPROVE/MAJOR
│   ├─ Judge + StageRouter:
│   │   APPROVE → GOAL_ACHIEVED → 停止
│   │   MAJOR + retry<3 + 非连续2次 → next=developer (回路)
│   │   MAJOR + retry≥3 OR 连续2次 → HARD_STOP
│   └─ Checkpoint: 保存
│
└─ 输出: JSON { verdict, rounds, history, gate_summary }
```

## A.4 组件全景图

```
CLI / Plugin Layer
    /dev-loop  /status  /checkpoint  /project:*  /init
    hooks: session-start  post-edit  stop  pre-tool  on-pr
        │
Orchestrator (主循环)
    ├─ Plan (Task DAG, validate, get_tasks_by_stage)
    ├─ StageRouter (next_stage 决定 + MAJOR 追踪)
    ├─ GuardrailChain (pre/post, 按 timing+stage 过滤)
    ├─ run_round() → Agent.execute() × N → TaskOutcome[]
    ├─ ConvergenceJudge (4 级)
    └─ CheckpointStore (SQLite WAL)
        │
Agent Layer
    BaseAgent.execute(task, ctx) → tool_use loop → TaskResult
    ├─ ArchitectAgent (只读, 产出 plan+batch_plan)
    ├─ DeveloperAgent (全工具, TDD)
    └─ CriticAgent (只读+验证, 产出 verdict)
        │
Infrastructure
    Tools / Gates(7) / Channels(checkpoint专用) / Guardrails(5)
    CancellationToken / SemanticEvaluator / ToolRegistry
```

## A.5 关键设计决策

| # | 决策 | 理由 | 业界参照 |
|---|------|------|---------|
| D1 | **Stage-sequenced, Agent→LoopState 隔离** | Agent 不直接写共享状态，Orchestrator 统一写入 | Actor model + LangGraph PregelLoop 每 superstep 集中 apply_writes |
| D2 | **Tasks 外部定义，ArchitectAgent 可自动补充** | 有明确 Task 时直接执行；无 Task 时 ArchitectAgent 的 batch_plan 自动生成 | AutoGen Task 定义 + Devin plan-act-observe |
| D3 | **MAJORRetryTracker + StageRouter 分离** | 回路计数独立于状态机，纯函数 next_stage 不追踪历史 | LangGraph recursion_limit (独立于 PregelLoop) |
| D4 | **Gate 全并行，无依赖等待** | ContractGate 独立运行，读本 Gate 自己的判断，不等待其他 Gate | CI 并行 job 模式 |
| D5 | **LoopState 仅 Orchestrator 写入** | Agent 返回 TaskOutcome → Orchestrator 提取 → 写入 state | Actor model + AutoGen MessageEnvelope |
| D6 | **Plugin = Bash 委托 `uv run ae`** | Python 控制流 + Claude Code UX | Claude Code plugin convention |

## B1. 数据模型

### B1.1 LoopState — 运行时状态容器

文件: `auto_engineering/engine/state.py` | 类型: `@dataclass` | 写入者: 仅 Orchestrator (Agent 不直接写入)

| # | 字段 | Python 类型 | 约束 | 默认值 | 语义 | Stage |
|---|------|------------|------|--------|------|-------|
| 1 | `requirement` | `str` | 1..4096, NOT NULL | `""` | 原始需求 | (构造时) |
| 2 | `round` | `int` | 0..N, NOT NULL | `0` | 当前轮次 (1-indexed) | — |
| 3 | `stage` | `Literal["","architect","developer","critic"]` | 枚举 | `""` | 当前阶段 | — |
| 4 | `plan` | `str \| None` | 0..65535 chars | `None` | architect 产出的 Markdown 计划 | architect |
| 5 | `file_list` | `list[str] \| None` | 0..200, 每元素 ≤256 | `None` | 目标文件路径 | architect |
| 6 | `batch_plan` | `list[dict] \| None` | 0..20, 每 dict: `{id,description,files}` | `None` | 分批策略 (自动生成 Task 用) | architect |
| 7 | `files_changed` | `list[str] \| None` | 0..200 | `None` | 实际修改的文件 | developer |
| 8 | `commit_hash` | `str \| None` | `[0-9a-f]{40}` | `None` | git commit SHA | developer |
| 9 | `test_results` | `dict \| None` | `{"passed":int,"failed":int,"errors":int,"skipped"?:int}` | `None` | 测试结果汇总 | developer |
| 10 | `verdict` | `Literal["APPROVE","MAJOR",""]` | 枚举 | `""` | critic 最终判定 | critic |
| 11 | `findings` | `list[dict] \| None` | 0..50, `{file,line?,issue,severity,suggestion?}` | `None` | 问题清单 | critic |
| 12 | `critic_feedback` | `str \| None` | 0..65535 | `None` | 反馈文本 | critic |
| 13 | `round_history` | `deque[RoundHistory]` | maxlen=50 | `deque()` | 历史记录 | — |
| 14 | `thread_id` | `str` | UUID v4, PK col1 | 自动生成 | 会话标识 | (构造时) |
| 15 | `majors_in_a_row` | `int` | 0..N, default=0 | `0` | 连续 MAJOR 计数 | — |
| 16 | `total_majors` | `int` | 0..N, default=0 | `0` | 累计 MAJOR 计数 | — |

**Agent→LoopState 写入协议**:
1. Agent 返回 `TaskResult(values=dict)` — 仅含该 Agent 角色的输出字段
2. Orchestrator 从 `TaskResult.values` 提取字段 → 写入 LoopState
3. 无 Agent 直接持有 LoopState 引用进行写入

### B1.2 Task 来源规范（修复 P0-2）

Tasks 有两种来源，优先级:

**来源 1: 外部定义 (优先)**。CLI `--tasks tasks.yml` 或默认 `.ae-tasks.yml`:
```yaml
tasks:
  - id: impl-login
    role: developer
    description: "实现 POST /login 接口，含参数校验和 session 创建"
    expected_output: "可工作的登录接口 + 单元测试"
    depends_on: []
    file_targets: ["src/auth/login.py", "tests/test_login.py"]
  - id: review-login
    role: critic
    description: "审查登录实现"
    depends_on: [impl-login]
```

**来源 2: ArchitectAgent 自动生成 (fallback)**。无 Tasks 文件时:
1. Round 1 Stage=architect 执行 → 产出 `batch_plan`
2. Orchestrator 从 `batch_plan` 生成 Task 列表:
   - 每个 `batch_plan` 条目 → 1 个 `developer` Task
   - 最后追加 1 个 `critic` Task (依赖所有 developer task)
   - Task.id = `batch_plan[i].id`, role = `"developer"`, description = `batch_plan[i].description`, file_targets = `batch_plan[i].files`
3. 生成的 Tasks 重新 `Plan.validate()`

**batch_plan 条目的字段** (ArchitectAgent 输出的一部分):
```
{ "id": str, "description": str, "files": list[str], "depends_on": list[str] }
```

### B1.3 CheckpointEnvelope — 持久化容器

文件: `auto_engineering/loop/state/__init__.py` | 类型: `pydantic.BaseModel`

| # | 字段 | Python 类型 | 约束 | 语义 |
|---|------|------------|------|------|
| 1 | `thread_id` | `str` | UUID v4, PK | 会话标识 |
| 2 | `round` | `int` | ≥0, PK | 轮次 |
| 3 | `step` | `int` | ≥0, PK | Round 内子步骤 (当前固定=1) |
| 4 | `status` | `Literal["running","completed","interrupted","failed"]` | NOT NULL | 运行状态 |
| 5 | `state_snapshot` | `dict` | NOT NULL | LoopState 字段快照 |
| 6 | `majors_in_a_row` | `int` | ≥0 | 连续 MAJOR 计数 (持久化) |
| 7 | `total_majors` | `int` | ≥0 | 累计 MAJOR 计数 (持久化) |
| 8 | `retry_counters` | `dict[str,int]` | key=stage | per-stage guardrail 重试计数 (持久化) |
| 9 | `channels` | `dict[str,object]` | NOT NULL | Channel 实例 (未来扩展) |
| 10 | `created_at` | `datetime` | UTC, auto | 时间戳 |

### B1.4 LoopState → CheckpointEnvelope 转换

```
_save_checkpoint(state, retry_counters) → thread_id:
  1. snapshot = {  # 提取所有 LoopState 字段
       "requirement", "round", "stage", "plan", "file_list",
       "batch_plan", "files_changed", "commit_hash", "test_results",
       "verdict", "findings", "critic_feedback",
       "round_history": [asdict(h) for h in state.round_history]
     }
  2. envelope = CheckpointEnvelope(
       thread_id=state.thread_id,
       round=state.round, step=1,
       status=_derive_status(state),
       state_snapshot=snapshot,
       majors_in_a_row=state.majors_in_a_row,
       total_majors=state.total_majors,
       retry_counters=dict(retry_counters),
       channels={k: ch.copy() for k, ch in self._channels.items()}
     )
  3. store.save(envelope) → 返回 thread_id

resume(thread_id, round, step) → tuple[LoopState, dict[str,int]]:
  1. envelope = store.load(thread_id, round, step)
  2. state = LoopState(**envelope.state_snapshot)
  3. state.thread_id = envelope.thread_id
  4. state.round_history = deque(envelope.state_snapshot.get("round_history",[]), maxlen=50)
  5. state.majors_in_a_row = envelope.majors_in_a_row
  6. state.total_majors = envelope.total_majors
  7. retry_counters = dict(envelope.retry_counters)
  8. return (state, retry_counters)
```

## B2. 接口契约

### B2.1 Orchestrator

```
Orchestrator.__init__(
    requirement: str,                # 原始需求
    tasks: list[Task] | None,        # Task 列表 (None=由 ArchitectAgent 自动生成)
    config: OrchestratorConfig,      # 配置
) → Orchestrator

Orchestrator.run(
    cancellation: CancellationToken | None = None
) → RunResult                         # { history: list[RoundHistory], verdict: Verdict, thread_id: str }
  # raises: ConflictError, AEError(CONFIG_ERROR)
```

### B2.2 StageRouter (替代 next_stage)

```
StageRouter.__init__(
    max_majors_in_a_row: int = 2,    # 连续 MAJOR 上限
    max_total_majors: int = 3,       # 累计 MAJOR 上限
) → StageRouter

StageRouter.next(
    current_stage: str,              # 当前 stage
    verdict: str,                    # critic 的 verdict ("APPROVE"|"MAJOR"|"")
    majors_in_a_row: int,            # 当前连续 MAJOR 计数
    total_majors: int,               # 当前累计 MAJOR 计数
) → StageDecision                    # { next_stage: str|None, should_stop: bool, stop_reason: str|None }

# 判定逻辑:
#   stage==""            → next="architect"
#   stage="architect"    → next="developer"
#   stage="developer"    → next="critic"
#   stage="critic":
#     verdict="APPROVE"  → next=None, should_stop=False (由 Judge 的 GOAL_ACHIEVED 触发)
#     verdict="MAJOR":
#       if majors_in_a_row >= max_majors_in_a_row OR total_majors >= max_total_majors:
#         → next=None, should_stop=True, stop_reason="MAJOR 回路超限"
#       else:
#         → next="developer"
#     verdict=""         → next=None (防御)
```

### B2.3 GuardrailChain (修复 timing+stage 过滤)

```
Guardrail:
    name: str
    timing: Literal["pre","post"]
    applies_to: Literal["architect","developer","critic"]
    check(stage: str, state: LoopState) → GuardrailResult

GuardrailChain.__init__(guardrails: list[Guardrail]) → GuardrailChain

GuardrailChain.check(
    timing: Literal["pre","post"],
    stage: str,
    state: LoopState
) → GuardrailResult
  # 逻辑:
  #   for g in self.guardrails:
  #     if g.timing != timing: continue       ← 过滤 1
  #     if g.applies_to != stage: continue    ← 过滤 2
  #     result = g.check(stage, state)
  #     if result.action != "pass": return result
  #   return GuardrailResult("pass","")
```

**"drop" 动作说明**: 4 态中 `drop` 保留给用户自定义 Guardrail 使用。5 个内置 Guardrail 均不使用 `drop`（使用 `block` 或 `retry`）。扩展 API 允许注册自定义 Guardrail，其 `check()` 可返回 `drop` 来静默丢弃输出并重跑。

### B2.4 Gate.run_gates (移除依赖等待)

```
run_gates(gates: list[Gate], project_root: Path) → dict[str, Verdict]:
    # 所有 Gate 并行启动 (无依赖等待)
    coros = [asyncio.to_thread(g.run, project_root) for g in gates]
    gathered = await asyncio.gather(*coros, return_exceptions=True)
    # 每个 Gate 独立运行，不等待其他 Gate
    # 异常 → Verdict.failed(str(exc))
```

### B2.5 其余接口 (BaseAgent, AgentRuntime, ConvergenceJudge, SemanticEvaluator, run_round, TaskExecutor)

与 Round 2 保持一致。

## B3. Stage 状态机

### B3.1 状态图

```
[idle] ──CMD-01──→ [architect] ──guardrail pass──→ [developer]
                       ↑                              │
                       │    ┌──────────────────────────┘
                       │    ↓
                       │ [critic] ←──guardrail pass───┘
                       │    │
                       │    ├─ APPROVE → [idle] (GOAL_ACHIEVED)
                       │    │
                       │    └─ MAJOR ──→ StageRouter 判定:
                       │         ├─ 未超限 → [developer] (注入 feedback)
                       │         └─ 超限   → [idle] (HARD_STOP)
                       │
                  [any] ──round≥max──→ [idle] (HARD_LIMIT)
                  [any] ──SIGINT────→ [idle] (checkpoint 保存)
```

### B3.2 转换表

| # | 从 | 到 | 条件 | 触发组件 |
|---|----|-----|------|---------|
| T1 | idle | architect | CMD-01 | Orchestrator |
| T2 | architect | developer | Guardrail(post) pass | StageRouter |
| T3 | developer | critic | Guardrail(post) pass | StageRouter |
| T4 | critic | idle | verdict=APPROVE | ConvergenceJudge |
| T5 | critic | developer | verdict=MAJOR, majors_in_a_row<2, total_majors<3 | StageRouter |
| T6 | critic | idle | verdict=MAJOR, (majors_in_a_row≥2 OR total_majors≥3) | StageRouter |
| T7 | any | idle | round ≥ max_iterations | ConvergenceJudge |
| T8 | any | idle | SIGINT / CancellationToken | Orchestrator |

### B3.3 MAJOR 计数更新 (在 Orchestrator.run() 内)

```
if stage == "critic":
    if verdict == "APPROVE":
        state.majors_in_a_row = 0  # 重置
    elif verdict == "MAJOR":
        state.majors_in_a_row += 1
        state.total_majors += 1
```

## B4. Agent 规格

与 Round 2 一致，补充以下关键约束:

### B4.0 Agent→LoopState 隔离 (修复 P1-3)

- Agent 不接收 LoopState 引用。Agent 接收 `TaskContext { requirement, plan?, critic_feedback?, task: Task }` — 只读上下文。
- Agent 返回 `TaskResult(values=dict)` — 仅含本 Agent role 的输出字段。
- Orchestrator 从 TaskResult.values 提取字段 → 写入 LoopState:
  - ArchitectAgent.values → `state.plan`, `state.file_list`, `state.batch_plan`
  - DeveloperAgent.values → `state.files_changed`, `state.commit_hash`, `state.test_results`
  - CriticAgent.values → `state.verdict`, `state.findings`, `state.critic_feedback`
- 同 Stage 内多个 DeveloperAgent 并行执行时，各自独立返回 TaskOutcome，Orchestrator 合并结果后写入 state

### B4.4 LLM 重试明确责任 (修复 P1-1)

- **BaseAgent 内部**: 单次 `create_message` 调用超时 → 在 BaseAgent.execute() 内重试 (≤3, exp backoff, 参数见 B12.2)
- **Orchestrator**: 不重试 LLM 调用。Agent 返回 TaskOutcome(status="failed") → Orchestrator 记录并继续
- **无双重重试**: 重试仅在 BaseAgent 层，Orchestrator 不包装重试

### B4.5 超时层级 (修复 P1-7)

| 层级 | 超时 | 说明 |
|------|------|------|
| 单次 Anthropic API 调用 | 120s | SDK timeout |
| 单次 API 调用 + 重试 | 120s × 3 + backoff ≈ 370s max | BaseAgent 内部 |
| 单次 tool_use loop | 最多 10 轮 LLM 调用 | 每轮 120s, 总计 ≤ ~20min |
| 单 Stage 总时间 | 无硬超时 | 由 max_iterations + Gate timeout 间接约束 |

## B5. Guardrail 规格

### B5.1 5 个 Guardrail

| # | name | timing | applies_to | 检查内容 | 失败动作 |
|---|------|--------|-----------|---------|---------|
| G1 | RequirementValid | pre | architect | requirement 非空, 1..4096, 非仅控制字符 | block |
| G2 | PlanExists | post | architect | state.plan 非空 AND file_list 非空 | retry(≤3) |
| G3 | GitDiffExists | post | developer | git diff HEAD~1..HEAD 非空 | retry(≤3) |
| G4 | TestsPass | post | developer | test_results.failed=0 AND errors=0 | retry(≤3) |
| G5 | GitClean | post | developer | git status --porcelain 为空 | block |

**注**: `drop` 动作保留给用户自定义 Guardrail (扩展 API)。5 个内置 Guardrail 均不使用。

### B5.2 Guardrail 响应处理

```
_handle_guardrail_result(result, stage, retry_counters) → "continue"|"stop"|"retry":
    switch result.action:
        case "pass":    return "continue"
        case "block":   return "stop"
        case "drop":    # 自定义 guardrail: 清空当前 stage 字段 + 重跑
            _clear_stage_fields(state, stage)
            retry_counters[stage] = retry_counters.get(stage, 0) + 1
            return "retry"
        case "retry":
            if retry_counters.get(stage, 0) >= 3:
                return "stop"
            retry_counters[stage] = retry_counters.get(stage, 0) + 1
            return "retry"
```

## B8. Orchestrator 主循环 (完整修复)

### B8.1 run() 完整流程

```
Orchestrator.run(cancellation?) → RunResult:

  ┌─ 1. 初始化 ───────────────────────────────────────────┐
  │ 若 tasks 为 None 或空 → 标记 auto_generate_tasks=True   │
  │   否则 → Plan(tasks).validate() → DAG+文件隔离          │
  │ state = LoopState(requirement=requirement,              │
  │                    thread_id=uuid4())                    │
  │ max_iter = judge.config.max_iterations                  │
  │ retry_counters: dict[str,int] = {}                      │
  │ stage_router = StageRouter()                            │
  └────────────────────────────────────────────────────────┘

  ┌─ 2. 主循环 (for round_id in 1..max_iter) ────────────┐
  │                                                        │
  │ 2a. cancellation?.check() → break                      │
  │                                                        │
  │ 2b. current_stage = state.stage                        │
  │     if current_stage is None or "":                     │
  │         current_stage = stage_router.next("","",0,0)   │
  │         state.stage = current_stage.next_stage          │
  │                                                        │
  │ 2c. round_tasks = []                                   │
  │     if auto_generate_tasks AND stage=="architect":     │
  │         round_tasks = [architect_task(requirement)]     │
  │     elif self._plan:                                    │
  │         round_tasks = self._plan.get_tasks_by_stage(   │
  │                          current_stage)                 │
  │       # get_tasks_by_stage 按 task.role == stage 过滤   │
  │                                                        │
  │ 2d. ★ Guardrail PRE ──────────────────────────────    │
  │     pre = guardrail_chain.check("pre", stage, state)   │
  │     pre_action = _handle_guardrail_result(pre, stage,  │
  │                    retry_counters)                      │
  │     if pre_action == "stop": break                      │
  │     if pre_action == "retry":                           │
  │         _clear_stage_fields(state, stage)               │
  │         continue  # 重跑同一 stage                      │
  │                                                        │
  │ 2e. ★ Agent 执行 ──────────────────────────────────   │
  │     round_result = await run_round(...)                 │
  │     # Orchestrator 从 round_result.outcomes 提取值:     │
  │     for outcome in round_result.outcomes:              │
  │         if outcome.status == "completed":              │
  │             _apply_outcome_to_state(state, outcome)    │
  │                                                        │
  │ 2f. ★ Guardrail POST ─────────────────────────────    │
  │     post = guardrail_chain.check("post", stage, state) │
  │     post_action = _handle_guardrail_result(post,       │
  │                     stage, retry_counters)              │
  │     if post_action == "stop": break                     │
  │     if post_action == "retry": continue                 │
  │                                                        │
  │ 2g. ★ Semantic 评估 ──────────────────────────────    │
  │     semantic = await semantic_evaluator(round_result)   │
  │     history_entry.semantic_satisfied = semantic        │
  │                                                        │
  │ 2h. state.round_history.append(history_entry)          │
  │                                                        │
  │ 2i. ★ MAJOR 计数更新 ─────────────────────────────    │
  │     if stage == "critic":                              │
  │         if state.verdict == "APPROVE":                 │
  │             state.majors_in_a_row = 0                  │
  │         elif state.verdict == "MAJOR":                 │
  │             state.majors_in_a_row += 1                 │
  │             state.total_majors += 1                    │
  │                                                        │
  │ 2j. ★★★ 先保存 Checkpoint ★★★ ───────────────────    │
  │     _save_checkpoint(state, retry_counters)            │
  │     # 在 Judge 前保存 — 确保最后一轮也有 checkpoint     │
  │                                                        │
  │ 2k. ★ Judge 判定 ─────────────────────────────────    │
  │     verdict = judge.evaluate(state.round_history)      │
  │     if verdict.should_stop:                            │
  │         return RunResult(state.round_history, verdict)  │
  │                                                        │
  │ 2l. ★ StageRouter 决定下一 Stage ────────────────     │
  │     decision = stage_router.next(                      │
  │         current_stage, state.verdict,                  │
  │         state.majors_in_a_row, state.total_majors      │
  │     )                                                  │
  │     if decision.should_stop:                           │
  │         return RunResult(state.round_history,          │
  │           Verdict.stop(4, decision.stop_reason))        │
  │     if decision.next_stage is None:                    │
  │         break  # 所有 stage 完成 (APPROVE)             │
  │                                                        │
  │     # Stage 推进                                       │
  │     if decision.next_stage == "developer":             │
  │         # MAJOR 回路 — 注入 feedback (不清空 state)     │
  │         pass  # developer 的 system_prompt 自动含       │
  │              # critic_feedback (B4.2)                  │
  │     else:                                              │
  │         # 正常推进 — 清空下一 stage 字段                │
  │         _clear_stage_fields(state, decision.next_stage) │
  │                                                        │
  │     state.stage = decision.next_stage                  │
  │     retry_counters[decision.next_stage] = 0            │
  │     state.round += 1                                   │
  │                                                        │
  │     # auto_generate: architect 产出 batch_plan → Task  │
  │     if auto_generate_tasks and stage=="architect"      │
  │        and state.batch_plan:                           │
  │         self._plan = _tasks_from_batch_plan(           │
  │             state.batch_plan, state.file_list)          │
  │         auto_generate_tasks = False                    │
  └────────────────────────────────────────────────────────┘

  ┌─ 3. 退出 ─────────────────────────────────────────────┐
  │ return RunResult(state.round_history,                   │
  │   Verdict.stop(4, f"max_iterations={max_iter}"))        │
  └────────────────────────────────────────────────────────┘
```

### B8.2 _apply_outcome_to_state (Agent→LoopState 写入)

```
_apply_outcome_to_state(state, outcome):
    role = outcome.task_role  # 来自 Task.role
    values = outcome.output   # TaskResult.values (dict)

    if role == "architect":
        state.plan = values.get("plan")
        state.file_list = values.get("file_list")
        state.batch_plan = values.get("batch_plan")
    elif role == "developer":
        state.files_changed = values.get("files_changed")
        state.commit_hash = values.get("commit_hash")
        state.test_results = values.get("test_results")
    elif role == "critic":
        state.verdict = values.get("verdict")
        state.findings = values.get("findings")
        state.critic_feedback = values.get("critic_feedback")
```

### B8.3 _tasks_from_batch_plan

```
_tasks_from_batch_plan(batch_plan, file_list) → Plan:
    tasks = []
    for batch in batch_plan:
        tasks.append(Task(
            id=batch["id"],
            role="developer",
            description=batch["description"],
            file_targets=batch.get("files", []),
            depends_on=batch.get("depends_on", [])
        ))
    # 追加 critic task
    tasks.append(Task(
        id="critic-review",
        role="critic",
        description="审查所有 developer 产出",
        depends_on=[t.id for t in tasks]
    ))
    return Plan(tasks=tasks, requirement=state.requirement)
```

### B11.4 并行 Task 写入安全 (修复 P1-3)

同 Stage 内多个 developer Task 并行执行:
1. 每个 Agent 返回独立的 `TaskOutcome` (含 `output: dict`)
2. `run_round()` 收集所有 outcomes → 返回 `RoundResult.outcomes: list[TaskOutcome]`
3. Orchestrator 在 `run_round()` 返回**后**，**串行**遍历 outcomes → 调用 `_apply_outcome_to_state()`
4. Agent 不持有 LoopState 引用 → 无竞态条件

对于多 developer 写入 `files_changed` 的合并: 后来的 TaskOutcome 覆盖先前的 (Last-Write-Wins)。若需要追加合并，由 Orchestrator 的 `_merge_outcomes()` 实现 (当前: 取最后一个 completed outcome 的值)。
