## 一、问题清单（P0/P1/P2）

### 严重度说明

| 级别 | 含义 | 处理优先级 |
|------|------|-----------|
| **P0** | 阻断 v2.3 生产可用 | 1 周内修 |
| **P1** | 重要，影响架构一致性 | 1 季度 |
| **P2** | 建议，长期架构清理 | v3.0 / 长期 |

### P0 阻断（2 项）

| # | 标题 | 依据 | 优化建议 |
|---|------|------|---------|
| **P0-I1** | `version_utils.get_new_channel_versions` 生产零引用（代码逻辑虚化） | grep 确认 0 生产引用，仅 5 个测试引用；Phase 2.3-B 实现 channel_versions 累加但 Phase 2.3-C 增量选择用 history 而非 channel_versions | 文件头标记 `⚠️ 当前未被生产代码使用`；迁移到 convergence.py 作为 private 函数；或删除 |
| **P0-II** | `ae dev-loop` 默认走 v1.0 engine，v2.0 Orchestrator 非首选路径 | cli.py _run_loop_engine 被 dev_loop 命令默认调用；仅 ANTHROPIC_API_KEY 存在时走 v2.0 | 反转默认值：`ae dev-loop` 默认 v2.0 Orchestrator；`--use-v1` fallback；无 key 时友好提示 |

### P1 重要（4 项）

| # | 标题 | 依据 | 优化建议 |
|---|------|------|---------|
| **P1-I** | gates/ 11 文件 1386 行 — Guardrail(v1.0) + Gate(v2.0) 双体系 | builtin.py(197) + guardrail.py(99) vs 7 Gate files(~120 each) | builtin.py 标记"冻结"；docs/gate-system.md + BEACON 决策 21 |
| **P1-II** | version_utils.py 44 行独立文件为 1 个函数 | 过度抽象 | 合并到 convergence.py 作为 `_get_new_channel_versions`（private） |
| **P1-III** | loop/__init__.py 导出 40 个符号 — API 膨胀 | 公共 API 含内部类型、枚举 | 缩减到核心 15-20 个符号；内部类型保持 `import module` 访问 |
| **P1-IV** | engine/ + loop/ 命名不统一（LoopEngine vs Orchestrator） | 旧引擎 LoopEngine, 新引擎 Orchestrator。CLI 统一入口但内部命名不一致 | CLI wrapper 内部命名对齐 |

### P2 建议（4 项）

| # | 标题 | 优化建议 |
|---|------|---------|
| **P2-I** | engine/ 长期归档计划 | v3.0 时删除 927 行 |
| **P2-II** | builtin.py + guardrail.py 标记 DEPRECATED | 合并到 runtime/ |
| **P2-III** | checkpoint.py 序列化去冗余 | `_serialize_state`/`_deserialize_state` vs Pydantic 模型重复 |
| **P2-IV** | state.py 687 行拆分 | Channel / LoopState / MetricsSnapshot 分文件 |
