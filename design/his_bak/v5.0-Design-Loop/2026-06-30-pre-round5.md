# PART A: 架构概述（人读）

## A.1 一句话定位

`ae dev-loop` 是 **Stage-sequenced Agent loop**：接收自然语言需求 + Task 列表，按 architect → developer → critic 三阶段串行推进，每阶段 Agent 驱动 LLM 工具调用，Guardrail + Gate 双层质量防护，SQLite Checkpoint 持久化，Claude Code Plugin 承载 UX。

## A.2 核心概念

| 概念 | 定义 | 类比 |
|------|------|------|
| **Stage** | 三阶段流水线的一步 (architect/developer/critic) | LangGraph Node |
| **Round** | 一个 Stage 的完整周期 = Guardrail → Agent → Gate → Judge → Checkpoint | LangGraph Superstep |
| **Task** | 工作单元, role/description/expected_output/depends_on/file_targets | AutoGen Task |
| **Guardrail** | 进程内确定性校验, 4 态 (pass/block/drop/retry), 按 timing+stage 过滤 | CrewAI GuardrailResult |
| **Gate** | subprocess 质量检查, pass/fail, 7 道并行 | CI Pipeline Stage |
| **StageRouter** | 决定下一 Stage, 综合 stage+verdict+MAJOR 计数 | LangGraph conditional edge |
| **Agent→LoopState 隔离** | Agent 不写 LoopState; Agent 返回 TaskOutcome, Orchestrator 提取写入 | Actor model |

## A.3 端到端流程

```
用户输入: /dev-loop "实现登录" [--tasks tasks.yml]

预检: ANTHROPIC_API_KEY, project state, uv
Tasks 来源:
  有 --tasks → 加载 yml → Plan.validate()
  无 --tasks → ArchitectAgent 的 batch_plan 自动生成 Task

Round 1: architect → Agent 分析需求 → 产出 plan + file_list + batch_plan
Round 2: developer → Agent × N 并行 TDD → commit
Round 3: critic → Agent 审查 → APPROVE (停) 或 MAJOR → developer (回路)
  回路上限: 连续 2 次 MAJOR 或累计 3 次 → HARD_STOP

输出: JSON { status, thread_id, rounds, verdict, gate_summary }
```

## A.4 关键设计决策

| # | 决策 | 理由 | 业界参照 |
|---|------|------|---------|
| D1 | Stage-sequenced, Agent→LoopState 隔离 | Agent 不写共享状态 | LangGraph apply_writes + Actor model |
| D2 | Task 来源双模式 (外部定义 / ArchitectAgent 自动生成) | 灵活适配有/无预定义 Task 的场景 | Devin plan-act-observe |
| D3 | StageRouter + MAJOR 计数分离 | 回路控制独立于状态机 | LangGraph recursion_limit |
| D4 | Gate 全并行 | 7 Gate 互不依赖, 串行浪费 | CI 并行 job |
| D5 | Plugin = Bash 委托 `uv run ae` | Python 控制流 + Claude Code UX | Claude Code plugin convention |
| D6 | Checkpoint 在 Judge 前保存 | 确保最后一轮也有快照 | LangGraph after_tick |

**Agent→LoopState 写入协议** (隔离):
1. Agent 接收只读 `TaskContext`, 不持有 LoopState 引用
2. Agent 返回 `TaskOutcome(output=dict)` — 仅含本 role 的输出字段
3. Orchestrator 从 `TaskOutcome.output` 提取 → 写入 LoopState (见 B8.2)
4. 多 Agent 并行: 各自返回独立 TaskOutcome, Orchestrator 串行合并

## B4. Agent 规格

### B4.1 ArchitectAgent

| 维度 | 值 |
|------|-----|
| **role** | `"architect"` |
| **system_prompt** | 角色: 技术架构师。职责: 分析需求, 产出可执行计划。必须先输出文件集预检 (files_needed/files_to_create/files_to_modify) 再进入 plan。设计原则: 最小化变更/可独立验证/明确边界/文件粒度 ≤5/批次 |

### B4.2 DeveloperAgent

|------|-----|
| **role** | `"developer"` |
| **system_prompt** | 角色: 开发者。TDD 三步: RED(写失败测试→确认FAIL)→GREEN(最少实现→确认PASS)→REFACTOR(保持绿)。不偏离 plan.file_list。一个 batch 一个 commit。commit 前测试全绿。失败不 skip/注释。若提供 critic_feedback, 末尾追加: "上一轮 Critic 反馈:\n{feedback}\n请针对性修复。" |
| **tools** | `[ReadFileTool, WriteFileTool, EditFileTool, RunBashTool, GitCommitTool, RunTestsTool]` |
| **forbidden_tools** | 无 (受 Bash sandbox + 文件沙箱限制) |
| **model** | `claude-sonnet-4-6` |
| **max_tokens** | 8192 |
| **temperature** | 0.2 |
| **output_schema** | `{files_changed: [str], commit_hash: str, test_results: {passed:int, failed:int, errors:int}}` |
| **critic_feedback 注入** | 若 TaskContext.critic_feedback 非空 → system_prompt 末尾追加 |

### B4.3 CriticAgent

| 维度 | 值 |
|------|-----|
| **role** | `"critic"` |
| **system_prompt** | 角色: 代码审查者。审查 5 维度: 正确性/测试覆盖/代码质量/接口契约/运行时正确性。判定: APPROVE(全部通过) 或 MAJOR(≥1 P0 或 ≥3 P1)。原则: Fresh Context(不看开发者推理)/具体问题(file:line+修复建议)/诚实(不确定标 P2) |
| **tools** | `[ReadFileTool, GitDiffTool, RunTestsTool]` (只读+验证) |
| **forbidden_tools** | `WriteFileTool, EditFileTool, RunBashTool, GitCommitTool` |
| **model** | `claude-sonnet-4-6` |
| **max_tokens** | 4096 |
| **temperature** | 0.1 |
| **output_schema** | `{verdict: APPROVE\|MAJOR, findings: [{file, line?, issue, severity: P0\|P1\|P2, suggestion?}], critic_feedback: str}` |

### B4.4 Agent 工具调用循环

```
execute(task, ctx) → TaskOutcome:
  1. 构造 messages: [system_prompt, user_msg(task.description)]
  2. loop_count = 0
  3. while loop_count < max_tool_calls:
    3a. response = await llm.create_message(messages, tools, tool_choice="auto")
    3b. switch response.stop_reason:
        case "end_turn": break
        case "tool_use":
          for each tool_use in response.tool_use_blocks:
            tool = registry.get(tool_use.name)
            if tool is None: → 返回 error tool_result
            if not authz_check(role, tool.name): → 返回 error tool_result
            try:
              result = tool.execute(tool_use.input)
            catch Exception as e:
              result = ToolResult(error=str(e))
            messages.append({"role":"user","content":[{"type":"tool_result","tool_use_id":tool_use.id,"content":result.output or result.error}]})
        case "max_tokens": 标记 warning, break
    3c. loop_count += 1
  4. 双层解析 response.content:
     Layer1: Pydantic OutputModel.model_validate_json() → 成功返回
     Layer2: regex ```json fence 提取 → json.loads → validate → 成功返回
     都失败 → TaskOutcome(status="failed", error="parse_failed")
```

**LLM 重试** (BaseAgent 内部):
- 单次 `create_message` 超时/rate_limit/network_error → retry ≤3, exp backoff
- 参数: base_delay=1s, max_delay=30s, multiplier=2.0, jitter=±25%
- Orchestrator **不额外重试** LLM 调用

## B5. Guardrail 规格

### B5.1 5 个 Guardrail

| # | name | timing | applies_to | 检查内容 | 失败动作 |
|---|------|--------|-----------|---------|---------|
| G1 | **RequirementValid** | `"pre"` | `"architect"` | state.requirement 非空, 1..4096, 非仅控制字符 | `block` |
| G2 | **PlanExists** | `"post"` | `"architect"` | state.plan 非空 AND state.file_list 非空 AND len≥1 | `retry`(≤3) |
| G3 | **GitDiffExists** | `"post"` | `"developer"` | `git diff HEAD~1..HEAD --numstat` 输出非空 | `retry`(≤3) |
| G4 | **TestsPass** | `"post"` | `"developer"` | state.test_results.failed==0 AND errors==0 | `retry`(≤3) |
| G5 | **GitClean** | `"post"` | `"developer"` | `git status --porcelain` 输出为空 | `block` |

**`drop` 动作**: 4 态中保留给用户自定义 Guardrail。5 个内置 Guardrail 均不使用。扩展 API 允许注册自定义 Guardrail 返回 `drop` 来静默丢弃输出并重跑。

### B5.2 _handle_guardrail_result

```
_handle_guardrail_result(result, stage, retry_counters) → "continue"|"stop"|"retry":
    switch result.action:
        case "pass":  return "continue"
        case "block": return "stop"
        case "drop":
            _clear_stage_fields(state, stage)
            retry_counters[stage] = retry_counters.get(stage, 0) + 1
            return "retry"
        case "retry":
            if retry_counters.get(stage, 0) >= 3:
                return "stop"
            retry_counters[stage] = retry_counters.get(stage, 0) + 1
            return "retry"
```

## B6. Gate 规格

### B6.1 7 道 Gate

| # | gate_name | 实现 | 判定 | 耗时 | 超时 |
|---|-----------|------|------|------|------|
| G1 | `safety` | 正则扫描源码, 匹配 API key/token/PEM/身份证/手机号/银行卡/DB URL | 命中 → failed | <5s | 30s |

### B6.2 run_gates (全并行)

```
run_gates(gates, project_root) → dict[str, GateVerdict]:
    if not gates or project_root is None: return {}
    coros = [asyncio.to_thread(g.run, project_root) for g in gates]
    gathered = await asyncio.gather(*coros, return_exceptions=True)
    results = {}
    for item in gathered:
        if isinstance(item, BaseException): continue
        name, verdict = item
        results[name] = verdict
    return results
    # 全并行, 无依赖等待. 总耗时 ≈ max(单个)
```

## B7. Orchestrator 主循环

### B7.1 run() 完整流程

```
Orchestrator.run(cancellation?) → RunResult:

  ┌─ 1. 初始化 ───────────────────────────────────────────┐
  │ auto_gen = (tasks is None or len(tasks)==0)             │

  │ state = LoopState(requirement=req, thread_id=uuid4())   │

  └────────────────────────────────────────────────────────┘

  ┌─ 2. 主循环 (for round_id in 1..max_iter) ────────────┐

  │ 2a. cancellation?.check() → break                      │
  │                                                        │

  │ 2d. ★ PRE Guardrail ──────────────────────────────    │

### B7.2 _apply_outcome_to_state

```
_apply_outcome_to_state(state, outcome):
    role = outcome.task_role       # from Task.role
    values = outcome.output or {}  # TaskOutcome.output (dict)

    if role == "architect":
        if "plan" in values:         state.plan = values["plan"]
        if "file_list" in values:    state.file_list = values["file_list"]
        if "batch_plan" in values:   state.batch_plan = values["batch_plan"]
    elif role == "developer":
        if "files_changed" in values: state.files_changed = values["files_changed"]
        if "commit_hash" in values:  state.commit_hash = values["commit_hash"]
        if "test_results" in values: state.test_results = values["test_results"]
    elif role == "critic":
        if "verdict" in values:         state.verdict = values["verdict"]
        if "findings" in values:        state.findings = values["findings"]
        if "critic_feedback" in values: state.critic_feedback = values["critic_feedback"]
```

```
_tasks_from_batch_plan(batch_plan, requirement) → Plan:
    tasks = []
    for batch in batch_plan:
        tasks.append(Task(

            role="developer",
            description=batch["description"],

## B8. 收敛判定

### B8.1 停滞检测

```
detect_stagnation(history, threshold, diff_ratio_threshold) → bool:
    if len(history) < threshold + 1: return False
    consecutive = 0
    for i in range(len(history)-1, 0, -1):
        curr_size = history[i].files_changed + history[i].lines_added + history[i].lines_removed
        prev_size = history[i-1].files_changed + history[i-1].lines_added + history[i-1].lines_removed
        if curr_size == 0 and prev_size == 0:
            ratio = 0.0
        else:
            ratio = abs(curr_size - prev_size) / max(curr_size, prev_size)
        if ratio < diff_ratio_threshold:
            consecutive += 1
            if consecutive >= threshold: return True
        else:
            consecutive = 0
    return False
```

### B8.2 --no-gates 行为

设置 `config.gates=None` → run_round 跳过 gate → `gate_results={}` → Judge Level 3 检查 `if latest.gate_results:` 为 False → 跳过质量门。系统降为 3 级收敛 (硬上限 + 停滞 + 语义)。这是设计意图。

## B9. 并发模型

### B9.1 三层架构

| 层 | 机制 | 内容 |
|----|------|------|
| L1 asyncio event loop | 主线程 | Orchestrator.run(), run_round() |
| L2 ThreadPoolExecutor | asyncio.to_thread | LLM sync 调用, Gate subprocess, git diff |
| L3 Subprocess | subprocess.run | Linter, type checker, test runner, build |

### B9.2 资源约束

| 资源 | 上限 | 机制 |
|------|------|------|
| 并发 Agent | 同 Stage 内 ≤20 task | Plan.get_tasks_by_stage() |
| 并发 Gate | 7 (全并行) | asyncio.gather |
| SQLite 连接 | 1 写 (WAL) | busy_timeout=5000ms |
| RoundHistory 内存 | 50 条 | deque(maxlen=50) |
| Checkpoint WAL | <10MB | 定期 VACUUM |
| LLM 并发请求 | 1 per Agent 实例 | Anthropic SDK 管理 |

### B9.3 并行 Task 写入安全

同 Stage 内多个 task 并行:
1. 每个 Agent 返回独立 TaskOutcome (含 output=dict)
2. run_round() 收集所有 outcomes → RoundResult
3. Orchestrator 在 run_round() 返回后, 串行遍历 outcomes → _apply_outcome_to_state()
4. Agent 不持有 LoopState → 无竞态条件
5. 多开发者写入 files_changed: 后来的覆盖 (Last-Write-Wins)。未来 _merge_outcomes() 做追加合并

### B9.4 Task 取消

```
CancellationToken.check() → raise AEError(TASK_CANCELLED) | None
安全取消点:
  - Round 开始前 (2a)
  - Stage 边界 (2l 之后)
  不在 Agent LLM 调用中插入取消 (等当前调用返回)
  不在 Gate subprocess 中插入取消 (等 timeout)
```

## B10. 错误分类与恢复

### B10.1 错误树 (A-F)

| 类别 | 子类 | 动作 | 重试 |
|------|------|------|------|
| **A. 配置** | API_KEY 缺失 / project state 缺失 / uv 缺失 / project_root 不存在 | 提示 + exit 1 (project_root → 降级) | — |
| **B. Guardrail** | requirement 空 → block; plan 空 → retry≤3; git diff 空 → retry≤3; tests 失败 → retry≤3; git 不干净 → block | 见 B5 | ≤3 |
| **C. Gate** | safety/lint/type/test/build fail → failed; 工具缺失 → failed; timeout → failed | Verdict 写入 gate_results | — |
| **D. LLM** | timeout/rate_limit/network → retry≤3 exp backoff; auth_error → abort; max_tokens → 降级; tool 异常 → error 回传 LLM | 见 B4.4 | ≤3 |
| **E. 中断** | SIGINT → checkpoint 保存 + exit 130; CancellationToken → 正常退出; CancelError → TaskOutcome cancelled | — | — |
| **F. 数据** | checkpoint 损坏 → dict fallback; schema mismatch → migrate; disk full → abort; SQLite busy → retry≤3 | 见 B7.5 | ≤3 |

### B10.2 LLM 重试参数

| 参数 | 值 |
|------|-----|
| base_delay | 1.0s |
| max_delay | 30.0s |
| multiplier | 2.0 |
| jitter | ±25% |
| max_retries | 3 |

`delay = min(base * multiplier^(retry-1) * random(0.75, 1.25), max_delay)`

### B10.3 降级路径

| 场景 | 降级 | 影响 |
|------|------|------|
| SafetyGate 工具缺失 | skip | 无 secret 检测 (pre-tool hook 独立 sandbox) |
| CoverageGate | 永远 skip | CI 独立检查覆盖率 |
| SemanticEvaluator 不可用 | None | 降为 3 级收敛 |
| Gate subprocess timeout | failed | Judge 可能误判 |
| checkpoint.save 失败 | abort | 本轮丢失, resume 从上轮 |
| post-edit hook 异常 | catch + exit 0 | 不影响主流程 |
