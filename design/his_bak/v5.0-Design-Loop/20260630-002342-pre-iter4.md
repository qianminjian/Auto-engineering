## 1. 产品定位

`auto-engineering v5.0` 是**工程化团队用的、在 agent 里跑的、多轮自治开发 loop**。

| 维度 | 描述 |
|------|------|
| 形态 | Claude Code plugin, 装一次, 任何项目用 |
| 启动 | `/init 工程化` (scaffold) + `/dev-loop <需求>` (loop) |
| 核心 | 多轮 (architect → developer → critic) 自治迭代, deterministic gates, persistent checkpoint |
| 范围 | 工程化团队, 跨项目, 跨 session, 跨工程师 |
| 非目标 | 半成品, 单文件 demo, 简化版, 重度 LLM 自评, 不可调试 |

## 2. 核心架构 (Hybrid Python + Agent)

```
┌──────────────────────────────────────────────────────────────┐
│ Claude Code Session                                           │
│                                                               │
│  用户: /dev-loop "实现用户登录功能"                           │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ auto-engineering v5.0 plugin                           │  │
│  │   commands/dev-loop.md (slash command body)            │  │
│  │     ↓ Bash delegation (uv run)                         │  │
│  │   ┌────────────────────────────────────────────────┐  │  │
│  │   │ Python LoopEngine                               │  │  │
│  │   │   while True:                                   │  │  │
│  │   │     tick()                                      │  │  │
│  │   │       → StageGraph.next_stage(state)            │  │  │
│  │   │         决定下一 stage (architect/dev/critic)    │  │  │
│  │   │     execute()                                   │  │  │
│  │   │       → AgentRuntime.execute(stage, state)       │  │  │
│  │   │         调 Claude API 决策 + 工具               │  │  │
│  │   │     guardrail()                                │  │  │
│  │   │       → 5 个 guardrail 中间件                   │  │  │
│  │   │     gate()                                     │  │  │
│  │   │       → 7 个 gate subprocess 验证               │  │  │
│  │   │     after_tick()                               │  │  │
│  │   │       → SQLite checkpoint.save()               │  │  │
│  │   │     judge()                                    │  │  │
│  │   │       → 4 级收敛判定                           │  │  │
│  │   │   loop until judge says stop                    │  │  │
│  │   └────────────────────────────────────────────────┘  │  │
│  │     ↑                                                    │  │
│  │   Claude (per-round: 读 status, 调工具, 看结果)          │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

**关键设计原则**:
- **Python 严格 while-True** 控制流 (LangGraph 借鉴, 业界共识)
- **Claude API** 每个 Stage 内做智能决策 (AutoGen 借鉴)
- **Deterministic gates** 不靠 LLM 自评 (CrewAI Guardrail + 自定义 Gate)
- **Agent 提供 UX** + **Python 提供引擎** — 严格分工, 不混

## 3. 3 Stage + 条件边反馈回路

```
architect (plan) → developer (impl) → critic (review)
                                     ├─ APPROVE → gate 验证 → end
                                     └─ MAJOR → developer (feedback loop)
```

### 3.1 Stage 定义

| Stage | agent_type | 输入 channel | 输出 channel | 工具 |
|-------|-----------|------------|------------|------|
| **architect** | architect | requirement | plan, file_list, contracts | read_file, search_code, list_dir |
| **developer** | developer | plan, critic_feedback | files_changed, commit_hash, test_results | read_file, write_file, edit_file, run_bash, git_commit |
| **critic** | critic | files_changed, commit_hash, contracts, plan | verdict (APPROVE/MAJOR), findings, critic_feedback | read_file, git_diff, run_tests |

### 3.2 反馈回路 (v5.0 关键设计)

`critic_feedback` channel 自动注入 `developer.description_template`:

```python
developer.description_template = (
    "按计划实现: {plan}\n"
    "上一轮审查反馈: {critic_feedback}"  # ← 关键: critic 输出自动反馈给 developer
)
```

当 `verdict == "MAJOR"`, StageGraph 走 conditional edge → developer 重新执行 (读新的 critic_feedback).

### 3.3 StageGraph 完整定义

```python
# StageGraph 是 LangGraph StateGraph 的简化 (去 channel 类型系统)
class StageGraph:
    def __init__(self):
        self.stages: dict[str, Stage] = {}        # 3 个 stage
        self.edges: dict[str, str] = {}            # 固定边
        self.conditional_edges: dict[str, ConditionSpec] = {}  # 条件边

    def build_dev_loop_graph() -> StageGraph:
        g = StageGraph()
        g.add_stage(Stage(name="architect", ...))
        g.add_stage(Stage(name="developer", ...))
        g.add_stage(Stage(name="critic", ...))
        g.add_edge("architect", "developer")
        g.add_edge("developer", "critic")
        g.add_conditional_edge("critic", lambda s: s.verdict, {
            "APPROVE": "__end__",
            "MAJOR": "developer",  # 反馈回路
        })
        g.set_start("architect")
        return g
```

## 4. 5 Guardrail + 7 Gate (双层保险)

### 4.1 5 个 Guardrail (in-process, deterministic, CrewAI/AutoGen 借鉴)

```python
# 5 个 Guardrail + 4 态 action (CrewAI 借鉴)
class GuardrailAction:
    PASS = "pass"     # 放行
    BLOCK = "block"   # 中止循环
    DROP = "drop"     # 静默丢弃 (AutoGen DropMessage 借鉴)
    RETRY = "retry"   # 重试当前 stage

class GuardrailChain:
    """中间件链. 每个 GuardrailHandler 返回 GuardrailResult."""
    def __init__(self, handlers: list[GuardrailHandler]):
        self.handlers = handlers

    async def on_execute_start(self, ctx, task) -> GuardrailResult:
        for h in self.handlers:
            r = await h.on_execute_start(ctx, task)
            if not r.success:
                return r
        return GuardrailResult(success=True, action="pass")

    async def on_execute_end(self, ctx, result) -> GuardrailResult:
        for h in self.handlers:
            r = await h.on_execute_end(ctx, result)
            if not r.success:
                return r
        return GuardrailResult(success=True, action="pass")
```

| Guardrail | 触发时机 | 失败 action |
|-----------|---------|------------|
| **RequirementGuardrail** | architect 入口 | drop (requirement 为空) |
| **PlanExistsGuardrail** | architect 出口 | block (plan 文件不存在) |
| **GitDiffExistsGuardrail** | critic 入口 | block (无 commit 可审) |
| **GitCleanGuardrail** | developer 出口 | block (有未提交变更) |
| **TestsPassGuardrail** | developer 出口 | retry (测试未通过) |

### 4.2 7 个 Gate (subprocess, deterministic, v5.0 严格化)

| Gate | 命令 | 通过条件 | 阻塞 |
|------|------|---------|------|
| **Safety** | `gitleaks detect --no-banner` | 0 leaks | 阻塞 |
| **Lint** | `ruff check .` | 0 errors | 阻塞 |
| **TypeCheck** | `mypy .` | 0 errors | 阻塞 |
| **Contract** | `design/contracts/*.md` 校验 | 一致性 | 阻塞 |
| **Test** | `pytest -q` | 100% pass | 阻塞 |
| **Coverage** | `pytest --cov --cov-fail-under=80` | ≥80% | 阻塞 |
| **Build** | `python -m build` (或 `npm run build`) | exit 0 | 阻塞 |

**关键约束**: **Gates 不通过 → 退回, 即使 Critic APPROVE**. 这是 5 + 7 = 12 层保险, 不靠 LLM 自评.

## 5. 4 级收敛判定

```python
class ConvergenceJudge:
    def evaluate(self, history: list[RoundHistory], max_rounds: int) -> Verdict:
        # 优先级 1: GOAL_REACHED (Critic APPROVE + gates 全过)
        if history[-1].verdict == "APPROVE" and all_gates_passed:
            return Verdict(stop=True, level=1, reason="goal_reached")

        # 优先级 2: HARD_STOP
        if len(history) >= max_rounds:
            return Verdict(stop=True, level=4, reason="hard_stop")

        # 优先级 3: PLATEAU (连续 2 轮无变化)
        if is_plateau(history, threshold=2):
            return Verdict(stop=True, level=2, reason="plateau")

        # 优先级 4: NO_PROGRESS (历史无新 task)
        if no_new_task_in_round(history):
            return Verdict(stop=True, level=3, reason="no_progress")

        return Verdict(stop=False, level=0, reason="continue")
```

## 7. LoopEngine 完整实现

```python
class LoopEngine:
    """v5.0 核心: while True: tick → execute → guardrail → gate → after_tick → judge.

    借鉴 LangGraph PregelLoop (简化, 去掉 superstep/channel 类型系统).
    """

    def __init__(self, graph: StageGraph, runtime: AgentRuntime,
                 guardrail_chain: GuardrailChain, gates: list[Gate],
                 checkpoint_dir: str = ".ae-checkpoints",
                 max_rounds: int = 10):
        self.graph = graph
        self.runtime = runtime
        self.guardrail_chain = guardrail_chain
        self.gates = gates
        self.checkpoint = Checkpoint(dir=checkpoint_dir)
        self.max_rounds = max_rounds
        self._state = LoopState()
        self._cancellation = CancellationToken()

    async def run(self, requirement: str, max_rounds: int | None = None) -> LoopResult:
        steps = max_rounds or self.max_rounds
        self._state.requirement = requirement
        self._cancellation = CancellationToken()

        while True:
            self._cancellation.check()
            if not self._tick(steps):
                break
            try:
                result = await self.runtime.execute(self._current_stage, self._state)
            except OutputDropped:
                continue
            except GuardrailBlockedError:
                break

            if not all(self._run_gates()):
                continue  # gate 不通过, 退回

            self._after_tick(result)
            verdict = self._judge()
            if verdict.should_stop:
                break

        return LoopResult(state=self._state, history=self.checkpoint.history)

    def _tick(self, max_rounds: int) -> bool:
        if self._state.round >= max_rounds:
            return False
        next_stage = self.graph.next_stage(self._state)
        if next_stage is None:
            return False
        self._current_stage = next_stage
        return True

    def _run_gates(self) -> list[bool]:
        """7 个 gate 串行 (因为是 deterministic subprocess)."""
        return [gate.run(self._state) for gate in self.gates]

    def _after_tick(self, result):
        self._state.round += 1
        self.checkpoint.save(self._state)

    def _judge(self) -> Verdict:
        return ConvergenceJudge().evaluate(
            self.checkpoint.history, self.max_rounds
        )
```

## 10. 业界对比 (v5.0 差异化)

| 维度 | LangGraph | AutoGen | CrewAI | Devin | **v5.0 plugin** |
|------|-----------|---------|--------|-------|-----------------|

| 控制流 | Python 显式 | Python 消息队列 | Python Process DSL | LLM 内部 | **Python while-True** (严格) |
| Stage 反馈 | 用户写 | GroupChat | context 传递 | Agent 自主 | **critic_feedback channel** (LangGraph 借鉴) |
| 确定性验证 | interrupt 人工 | InterventionHandler | Guardrail 4 态 | LLM 自评 | **5 Guardrail + 7 Gate 双层** |
| 持久化 | SqliteSaver | Memory | 自带 | 无 | **SQLite + thread_id 隔离** |
| 恢复 | checkpoint id | topic id | resume index | 无 | **thread_id resume** |

**v5.0 核心差异化**:
1. **Plugin 形态** — 业界 loop 都是 Python 库, 我们跑在 Claude Code agent 里
2. **双层保险** — 5 Guardrail + 7 Gate = 12 层验证, 业界大多 LLM 自评
3. **完整 init** — 8 类型 × 4 语言 = 32 模板, 业界 loop 工具都没有
4. **反馈回路显式** — critic_feedback channel 自动注入 developer 输入

## 11. 不做的事 (Anti-patterns)

- ❌ 不让 Claude 跑 while-True (控制流在 Python)
- ❌ 不让 LLM 自评替代 Gate (BEACON §Gate 5 决策)
- ❌ 不在 plugin 里重写 LoopEngine (v5.0 保留 Python 引擎)
- ❌ 不重写 jinja2 模板渲染 (Copier 模式已验证)
- ❌ 不在 init 时启用 9 个选择 (v1.0 决策, defaults-first 解决)
- ❌ 不"半成品 hook" (5 个全部按 v1.0 规划, 不缩为 2)
- ❌ 不"半成品 commands" (8 个完整, 不缩为 3)
- ❌ 不删 CLI (CI / 脚本仍用 `ae` 命令)

**位置**: `skills/auto-engineering/SKILL.md`

```markdown
---
name: auto-engineering
description: |
  Team-grade loop engineering + multi-agent collaboration. Loaded when user invokes

# auto-engineering v5.0 — Loop Engineering Protocol

## 协议 (v5.0 Loop Engineering 完整)

### 1. LoopEngine 跑在 Python (不靠 LLM 自组织)
- **控制流在 Python** (LangGraph PregelLoop tick/after_tick 借鉴, v1.0 设计哲学)
- **Python 严格 while-True** 调度
- **Claude API** 在每个 Stage 内做智能决策 (AutoGen 借鉴)
- **Agent 提供 UX** (Claude Code) + **Python 提供引擎** (LoopEngine) — 严格分工, 不混

### 2. 3 Stage + 条件边反馈回路 (LangGraph 借鉴)

```
architect (plan) → developer (impl) → critic (review)
                                     ├─ APPROVE → gate 验证 → end
                                     └─ MAJOR → developer (feedback loop, critic_feedback channel)
```

StageGraph 完整定义见主设计文档 §3.3.

### 3. 5 Guardrail (CrewAI + AutoGen 借鉴, in-process 确定性)

- RequirementGuardrail (architect 入口, drop if empty)
- PlanExistsGuardrail (architect 出口, block if no plan file)
- GitDiffExistsGuardrail (critic 入口, block if no commit)
- GitCleanGuardrail (developer 出口, block if uncommitted)
- TestsPassGuardrail (developer 出口, retry if fail)

### 4. 7 Gate (subprocess 确定性验证, 不靠 LLM 自评)

safety → lint → type_check → contract → test → coverage → build.
**Gates 不通过 → 退回, 即使 Critic APPROVE** (BEACON §Gate 5 决策).

### 5. 4 级收敛判定 (ConvergenceJudge)

| Level | Trigger | Action |
|-------|---------|--------|
| GOAL_REACHED | Critic APPROVE + gates 全过 | Stop |
| HARD_STOP | round ≥ max_rounds | Stop |
| PLATEAU | 连续 2 轮 diff < 5% | Stop |
| NO_PROGRESS | 历史无新 task | Stop |

### 6. 状态持久化 (SQLite + thread_id 隔离)

- LoopState (跨 Stage 传值, Pydantic)
- Checkpoint (SQLite, WAL 模式, thread_id 隔离)
- Resume 流程: thread_id → 读最后 state → 注入 LoopEngine.checkpoint

### 7. Claude (plugin) vs Python (LoopEngine) 分工

| 谁 | 做什么 |
|---|--------|
| **Claude (plugin UX)** | 调 /dev-loop, 读 status, 调文件工具, 看结果 |
| **Python (LoopEngine)** | while-True 调度, StageGraph 路由, Guardrail, Gate, checkpoint |

## Claude 在 plugin 里的实际操作 (skill 加载后)

当用户调 `/dev-loop "<需求>"`:

1. **预检** (Bash): ANTHROPIC_API_KEY 是否设, .ae-answers.yml 是否存在
2. **Bash 委托**: `uv run ae dev-loop "$ARGUMENTS"` (阻塞, Python 引擎跑)
3. **后处理** (Bash): `uv run ae status` 看最终结果
4. **响应用户**: 把 verdict 翻译为自然语言 (GOAL_REACHED / HARD_STOP / PLATEAU / NO_PROGRESS)

## 不要做 (Anti-patterns)

- ❌ 不在 Claude 里手动模拟 LoopEngine (Python 的工作, Claude 不要重抄)
```

### 19.2 `hooks/post-edit.sh` (PostToolUse: Edit|Write|MultiEdit)

```bash
#!/bin/bash
# v5.0 PostToolUse Hook: Edit/Write 后跑 SafetyGate 扫 secrets
# 触发: Claude Code Edit/Write 工具后
# 超时: 30 秒
# 失败处理: gate 失败输出 warning, 不阻塞 (warning 给 Claude 看, 它能 fix)

set -uo pipefail

# 1. 跳过非代码文件 (避免不必要的 scan)
case "$CLAUDE_TOOL_INPUT_FILE_PATH" in
  *.py|*.env|*.yml|*.yaml|*.json|*.ts|*.tsx|*.js|*.jsx|*.sh|*.tf|*.toml)
    # 继续执行
    ;;
  *)
    # 不是需要扫的代码/配置/环境文件, 跳过
    exit 0
    ;;
esac

# 2. PATH 含空格 / cwd 异常 / uv 不存在 — 防御性检查
if [ ! -d ".ae-checkpoints" ] && [ ! -f ".ae-answers.yml" ]; then
  # 未初始化项目, 不跑 gate
  exit 0
fi

if ! command -v uv >/dev/null 2>&1; then
  echo "[AE PostEdit] uv 未找到, 跳过 SafetyGate"
  exit 0
fi

# 3. 调 SafetyGate (超时 25 秒, 留 5 秒 buffer)
output=$(timeout 25 uv run python -c "
from auto_engineering.gates import SafetyGate
from pathlib import Path
import sys
try:
    v = SafetyGate().run(Path('.'))
    print(v.message)
    sys.exit(0 if v.passed else 1)
except Exception as e:
    print(f'gate error: {e}')
    sys.exit(0)  # gate 异常不阻塞
" 2>&1)
exit_code=$?

# 4. 输出结果 (不阻塞, 只 warn)
if [ $exit_code -ne 0 ]; then
  echo "[AE PostEdit] ⚠️ SafetyGate 警告: $output" >&2
  echo "[AE PostEdit] 建议: 检查 secret 是否泄漏, 修复后 /status 看"
fi

exit 0  # 永远不阻塞 Edit
```

### 19.3 `hooks/stop.sh` (Stop)

```bash
#!/bin/bash
# v5.0 Stop Hook: Session 停止时自动 save checkpoint
# 触发: Claude Code session 正常停止
# 超时: 10 秒
```

### 19.4 `hooks/pre-tool.sh` (PreToolUse: Bash)

```bash
#!/bin/bash
# v5.0 PreToolUse Hook: Bash 调用前检查 shell 黑名单
# 触发: Claude Code Bash 工具调用前
# 超时: 5 秒 (必须快, 阻塞 user 体验)
# 退出码 0 放行, 退出码 2 拒绝

set -uo pipefail

# 1. 只拦截 Bash 工具 (其他工具放过)
if [ "${CLAUDE_TOOL_NAME:-}" != "Bash" ]; then
  exit 0  # 不是 Bash 工具, 放行
fi

# 2. 取命令 (兼容不同 Claude Code 版本的 env 变量名)
COMMAND="${CLAUDE_TOOL_INPUT_COMMAND:-${CLAUDE_BASH_COMMAND:-}}"
if [ -z "$COMMAND" ]; then
  exit 0  # 拿不到命令, 放行 (避免误拦截)
fi

# 3. 黑名单检查 (借鉴 v1.0 P1-S2 + v2.5 P1-S2 扩展)
case "$COMMAND" in
  # 危险: 反向 shell / 远程执行
  *"|sh"*|*"|bash"*|*"|zsh"*|*"nc -e"*|*"ncat -e"*|*"bash -i"*"/dev/tcp/"*)
    echo "[AE PreTool] BLOCKED: shell pipe / reverse-shell pattern detected" >&2
    echo "[AE PreTool] 命令: ${COMMAND:0:80}..." >&2
    exit 2
    ;;
  # 危险: Python 任意执行 (绕过安全审计)
  *"python -c"*|*"python3 -c"*|*"python -c "*|*"python3 -c "*)
    echo "[AE PreTool] BLOCKED: 'python -c' 不允许 (用脚本文件)" >&2
    echo "[AE PreTool] 命令: ${COMMAND:0:80}..." >&2
    exit 2
    ;;
  # 危险: eval 命令替换
  *"eval "*|*"eval$"*|*"$(("*)
    echo "[AE PreTool] BLOCKED: eval / 命令替换不允许" >&2
    exit 2
    ;;
  # 危险: 网络下载并执行
  *"curl "*|*"wget "*|*"fetch "*)
    if echo "$COMMAND" | grep -qE "(\| ?(sh|bash|zsh|python)|\>.*(sh|bash))"; then
      echo "[AE PreTool] BLOCKED: download-and-exec pattern (curl|fetch > script.sh)" >&2
      exit 2
    fi
    ;;
  # 危险: 危险文件系统操作
  *"rm -rf /"*|*"rm -fr /"*|*"mkfs"*)
    echo "[AE PreTool] BLOCKED: 危险 FS 命令" >&2
    exit 2
    ;;
  # 危险: 写敏感文件
  *"> /etc/*"*|*"tee /etc/*"*|*"dd if="*|*"chmod 777 /etc"*)
    echo "[AE PreTool] BLOCKED: 写 /etc 或破坏性命令" >&2
    exit 2
    ;;
esac

# 4. 放行
exit 0
```

## 22. 跨平台注意 (v5.0 实际工程化必须)

### 22.1 macOS vs Linux vs Windows

| 方面 | macOS | Linux | Windows |
|------|------|-------|---------|
| `date` 命令 | BSD (无 `-d` 格式) | GNU | 无 (PowerShell) |
| `realpath` | macOS 13+ 自带 | 通常自带 | 无 (用 `cygpath`) |
| `timeout` | 自带 (coreutils) | 自带 | 无 (用 `Start-Process -TimeoutSec`) |
| `case` | 兼容 | 兼容 | 不兼容 (用 `if/else`) |
| `uv` | 可装 | 可装 | 可装 (WSL 或 native) |
| `sqlite3` | 自带 | 自带 | 需装 |
| `git` | 自带 | 自带 | 自带 (Git for Windows) |

**v5.0 跨平台策略**:
- **hooks**: 全部用 `bash` (POSIX), 不依赖 GNU-only 选项
- **Python 引擎**: 跨平台, 但 tests/ 假设 Linux/macOS (Windows 测试是 best-effort, CI 跑 Linux)
- **path 引用**: 用 `pathlib.Path`, 不用 `os.path.join` (跨平台)
- **环境变量**: 大写 + snake_case (符合 POSIX + Windows env var 约定)

### 22.2 PATH 含空格 (防御)

**问题**: 用户项目可能在 `/Users/Some User/My Project/` (含空格). 一些 shell 操作 (尤其 eval) 会断.

**v5.0 防御**:
- hooks 用 `"$VAR"` 双引号包裹所有变量
- 不用 `eval` 解析 JSON (用 `python -c`)
- 不假设 cwd 无空格 (用 `$(pwd)` 而非 `pwd`)

### 22.3 Windows 兼容性 (v5.0 限制)

**v5.0 设计**: **Windows 不在 v5.0 支持范围**. 原因:
- Bash hooks 在 Windows 需 WSL/Git-Bash
- SQLite WAL 在 Windows 行为有差异
- 测试 CI 跑 Linux (macOS 是 best-effort)

**未来 v6.0**: 若需求明确, 加 Windows 兼容层 (PowerShell hooks + Win32 path)

### 29.2 关键架构决策记录 (v5.0 新增)

| # | 决策 | 理由 | 替代方案 |
|---|------|------|---------|
| v5-1 | **plugin 薄包装**, 不重写 v2.5 引擎 | 564 测试是契约, 重写 = 烧桥 | 重写 LoopEngine (否, 风险高) |
| v5-2 | **Bash 委托 `uv run ae <cmd>`** (而非 MCP server) | Bash 是 Claude Code 原生工具, 0 依赖 | MCP server (需额外 runtime) |
| v5-3 | **plugin.json 4 个 hook 事件** (SessionStart, PostToolUse, Stop, PreToolUse) | 覆盖 init/dev-loop/生命周期, 不漏 | 加 on-pr/on-issue (事件支持不成熟) |
| v5-4 | **commands 全部 8 个都在 v5.0** (不缩为 3-4 个) | v1.0 渐进启用哲学, 7 sub-commands + dev-loop + init = 完整 UX | 缩为 3-4 个 (v1.0 9 选择疲劳不可逆) |
| v5-5 | **hooks 全是 shell (POSIX)**, 不依赖 GNU-only 选项 | 跨平台 (macOS/Linux), 简洁 | Python hooks (重, 不需要) |
| v5-6 | **plugin 装失败 fallback 到 CLI** (硬约束) | CLI 完全独立, 永远可用, plugin 是 UX 增强 | plugin 是必需 (会断裂 v2.5) |
| v5-7 | **SKILL.md 1 个主 skill** (不 5 个分立) | 借鉴 gsd-* 模式, 1 个中心, 协议集中 | 5 个原 v1.0 分立 (碎片化, 读者迷失) |
| v5-8 | **3 Stage 反馈回路显式 (critic_feedback channel)** | 不靠 LLM 隐式传, 状态对象化 | Agent 自主反馈 (不可靠) |
| v5-9 | **5+7=12 层保险** (5 Guardrail + 7 Gate) | Gate 是 deterministic subprocess, 不可绕过 | LLM 自评 (不可靠, 业界默认) |
| v5-10 | **WAL 模式 + thread_id 隔离** | 并发读不互斥写, 多 session 隔离 | 无 WAL (单 session) |

### 29.3 数据流 (LoopEngine 一轮)

```
[Round N 开始]
       ↓
┌─ tick() ────────────────────────────────────────────────┐
│ 1. self._state.round < max_rounds?                       │
│ 2. next_stage = self.graph.next_stage(self._state)        │
│    - architect (round 0) → developer (round ≥1)         │
│    - developer → critic                                  │
│    - critic: verdict==APPROVE → __end__                  │
│    - critic: verdict==MAJOR → developer (反馈!)          │
│ 3. self._current_stage = next_stage                       │
│ 4. self._state.stage = next_stage.name                    │
└──────────────────────────────────────────────────────────┘
       ↓
┌─ execute() ──────────────────────────────────────────────┐
│ ┌─ on_execute_start (5 Guardrail) ──────────────────────┐│
│ │  RequirementGuardrail → drop if requirement==""     ││
│ │  PlanExistsGuardrail (architect 出口)               ││
│ │  GitDiffExistsGuardrail (critic 入口)               ││
│ │  GitCleanGuardrail (developer 出口)                ││
│ │  TestsPassGuardrail (developer 出口)                ││
│ └────────────────────────────────────────────────────────┘│
│ → AgentRuntime.execute(stage, state):                    │
│   - 调 AnthropicProvider.create_message                  │
│   - Agent.tool_use loop (调工具: Read/Write/Bash/Git)  │
│   - 返回 StageResult (含 output_channels)               │
│ ┌─ on_execute_end (5 Guardrail) ────────────────────────┐│
│ │  PlanExistsGuardrail (architect 出口)               ││
│ │  GitDiffExistsGuardrail (critic 入口)               ││
│ │  GitCleanGuardrail (developer 出口)                ││
│ │  TestsPassGuardrail (developer 出口)                ││
│ └────────────────────────────────────────────────────────┘│
│ → 若 Guardrail 失败 (BLOCK/DROP/RETRY):                   │
│   - DROP: continue (静默丢弃)                          │
│   - BLOCK: break loop (中止)                            │
│   - RETRY: continue (重试当前 stage)                    │
└──────────────────────────────────────────────────────────┘
       ↓
┌─ gates() ─────────────────────────────────────────────────┐
│ safety → lint → type_check → contract → test → coverage → build │
│ 任何 gate 失败 → continue (退回重做)                  │
│ 全过 → 进入 after_tick                                │
└──────────────────────────────────────────────────────────┘
       ↓
┌─ after_tick() ─────────────────────────────────────────────┐
│ 1. self._state.round += 1                               │
│ 2. self._state.apply_writes(result)                      │
│ 3. self.checkpoint.save(self._state)                     │
│    → SQLite thread_id + round + state_json              │
│ 4. 更新 channel_versions                                │
└──────────────────────────────────────────────────────────┘
       ↓
┌─ judge() ────────────────────────────────────────────────┐
│ 1. GOAL_REACHED? → critic_verdict==APPROVE && all_gates  │
│ 2. HARD_STOP? → round ≥ max_rounds                     │
│ 3. PLATEAU? → 连续 2 轮 diff < 5%                      │
│ 4. NO_PROGRESS? → 历史无新 task                        │
│ 5. 否则 → continue (继续下一轮)                       │
└──────────────────────────────────────────────────────────┘
       ↓
[Round N+1 或 end]
```

## 34. Troubleshooting (常见问题 + 解法)

> 按用户"在原文件增加"规则放这里, 不生成独立文件.

### 34.1 plugin 装失败

| 症状 | 原因 | 解法 |
|------|------|------|
| `claude plugin install` 报 "permission denied" | plugin 目录权限 | `chmod -R u+r /path/to/plugin` |
| `claude plugin install` 报 "not a valid plugin" | plugin.json 解析错 | `python -c "import json; json.load(open('plugin.json'))"` 验证 |
| plugin 装了但不工作 (slash command 找不到) | 路径错 (相对路径 vs `$PLUGIN_PATH`) | 改用 `$PLUGIN_PATH/hooks/...` 绝对引用 |
| SessionStart hook 没触发 | hook 文件无 +x 权限 | `chmod +x hooks/*.sh` |
| hook 报 "timeout" 但实际不慢 | timeout 值设太小 | 改 plugin.json 的 timeout (例如 30s 改 60s) |

### 34.2 dev-loop 跑不起来

| 症状 | 原因 | 解法 |
|------|------|------|
| `uv run ae dev-loop` 报 "ANTHROPIC_API_KEY not set" | 环境变量未设 | `export ANTHROPIC_API_KEY=sk-ant-...` 或 `echo "ANTHROPIC_API_KEY=..." >> .env` |
| 报 "No such file or directory: .ae-answers.yml" | 项目未初始化 | 先 /init 或 `uv run ae init . --force --defaults` |
| Round 1 后 LLM 报 rate limit | API key 配额不足 | 升级 Anthropic plan 或用 haiku 模型 (cheap) |
| 报 "gitleaks not found" | SafetyGate 依赖缺失 | `brew install gitleaks` (macOS) / `apt install gitleaks` (Linux) |
| 报 "ruff: command not found" | Lint gate 依赖缺失 | `pip install ruff` 或 `uv add --dev ruff` |
| 报 "mypy: can't read config" | mypy 配置错 | 修 `pyproject.toml [tool.mypy]`, 或 dev-loop 跳过 mypy (改 gate.py) |
| 单轮跑 30+ 分钟 | 网络慢 / API 慢 | 增大 round timeout, 改用 claude-haiku-4-5 (快 5x) |

### 34.3 dev-loop 行为异常

| 症状 | 原因 | 解法 |
|------|------|------|
| critic 总是 APPROVE (没真正审查) | critic system_prompt 弱 | 检查 `auto_engineering/agents/prompts.py` CRITIC_SYSTEM_PROMPT, 强化 6 维评审 |
| critic 总是 MAJOR (太严) | critic system_prompt 严 | 调整 prompt, 接受"次优但可工作"的方案 |
| round 永远不停止 (NO_PROGRESS 不触发) | history 没记录新 task | 检查 `LoopEngine._after_tick()` 是否正确写入 history |
| PLATEAU 一直触发 (连续 2 轮 diff < 5%) | 任务太小, round 太短 | 增大 `is_plateau()` 的 threshold (如 5% → 10%) |

### 34.4 跨 session 状态丢失

| 症状 | 原因 | 解法 |
|------|------|------|
| `/checkpoint resume` 报 "checkpoint not found" | thread_id 错 | 跑 `/checkpoint list` 看真实 id |
| 状态文件损坏 (SQLite 损坏) | 磁盘满 / 异常断电 | 删除 `.ae-checkpoints/store.db`, 重新 /init (历史丢失) |
| SessionStart hook 输出空 | cwd 不在 git repo | 在有 `.ae-answers.yml` 的目录启动 Claude Code |

### 34.5 Plugin 与 v2.5 CLI 冲突

| 症状 | 原因 | 解法 |
|------|------|------|
| `/init` 跑了 2 次, 文件被覆盖 | plugin 和 CLI 都调 | /init 完成后, 不要再 `uv run ae init` |
| `.ae-answers.yml` 内容被改 | 二次 init 重新生成 | 备份 .ae-answers.yml 后 init, 或 `ae init --from-answers` 复用 |
