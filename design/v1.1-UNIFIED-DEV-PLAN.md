## 七、风险评估

### 7.1 高风险

1. **LLM 真实调用的成本控制**:每次 dev-loop 消耗数千 token,必须 budget 限制
2. **Agent 输出解析稳定性**:LLM 输出格式变化,parser 双层防御仍可能失败
3. **Tools 真实执行的副作用**:WriteFile/Bash 可能破坏真实文件系统(必须路径白名单)
4. **Phase 5 v2.0 Channel 改造破坏 v1.0**:Channel 是基础抽象,必须向后兼容

### 7.2 中风险

1. **A1 增量模式设计**:scaffold.py 增量逻辑复杂,可能引入新偏差
2. **ruff B904 修复影响异常处理**:`raise X from e` 改变异常链,需逐个判断
3. **CancellationToken + LLM API 中断**:LLM 阻塞时 Ctrl-C 不能立即停止
4. **真实 LLM 端到端测试**(D3.02):需 ANTHROPIC_API_KEY,成本不可控

### 7.3 低风险

1. CLI 命令扩展(标准模式)
2. Tools registry 字符串解析(简单)
3. 文档同步(模板化操作)
