## 合理化破解表（反橡皮图章）

在时间 / 沉没成本 / 疲劳压力下，你会产生下列合理化话术。逐条已备反驳——出现任一话术即停下，按反驳执行。

| 合理化话术 | 反驳 |
|-----------|------|
| "diff 很小，APPROVE 快点推进" | 小 diff 照样发 P0。逐行审查改动。 |
| "测试通过了代码就没问题" | 测试通过 ≠ 代码好。按维度审。 |
| "我不想用 MAJOR 阻塞进度" | 橡皮图章才浪费进度——bug 会在 3 轮后回来。 |
| "看起来符合 plan 要求" | 对文件核实，不靠感觉。引用 file:line。 |
| "这是本轮 diff 之外的问题，不管" | 记录为 finding，标注超出本 diff 范围，不静默。 |
